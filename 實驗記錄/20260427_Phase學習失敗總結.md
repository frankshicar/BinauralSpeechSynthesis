# HybridPhysicalLearned 訓練分析

## 問題：Phase 下降幅度太小

### 訓練進度 (Epoch 1-11)

```
Epoch 1:  IPD: 0.9633, Phase_L: 0.9993, Phase_R: 0.9992
Epoch 4:  IPD: 0.9493, Phase_L: 0.9993, Phase_R: 0.9990
Epoch 11: IPD: 0.9148, Phase_L: 0.9992, Phase_R: 0.9993
```

### 觀察

1. **IPD 有改善**：0.9633 → 0.9148 (下降 5%)
2. **Phase 幾乎不動**：0.9993 → 0.9992 (幾乎沒變)
3. **Waveform L2 卡住**：0.000898 → 0.000830 (改善 7.5%)

### 問題分析

#### 1. Loss 權重不平衡

```python
loss = 1.0 × Waveform_L2    # 0.000830 (量級 10^-4)
     + 0.5 × Magnitude      # 0.052    (量級 10^-2)
     + 1.0 × IPD            # 0.9493   (量級 10^0)
     + 0.01 × ITD_reg       # 0.000000 (量級 10^-6)
```

**實際梯度貢獻**：
```
IPD >> Magnitude >> Waveform >> ITD_reg
```

**問題**：
- IPD loss 主導訓練
- Waveform L2 貢獻太小
- Phase 沒有直接優化目標

#### 2. 物理 ITD 太強

```python
total_ITD = physical_ITD + residual_ITD
```

**觀察**：
- ITD_reg = 0.000000 → residual_ITD ≈ 0
- 模型完全依賴物理 ITD
- 沒有學習任何修正

**問題**：
- 物理模型可能不夠準確
- 但網路不敢修正（正則化太強）
- 失去了學習的意義

#### 3. Phase 沒有直接約束

```python
# 當前只監控，不優化
Phase_L loss: 0.9992
Phase_R loss: 0.9993
```

**問題**：
- Phase 不在 loss 中
- 只通過 Waveform L2 間接影響
- 梯度信號太弱

### 根本原因

**這個方法仍然無法解決 Phase 學習問題**：

1. **物理 ITD 只是低頻近似**
   - 只對 <1kHz 準確
   - 高頻 Phase 仍然隨機
   - 無法捕捉複雜的 HRTF

2. **Residual ITD 被過度正則化**
   - λ_itd_reg = 0.01 太大
   - 網路不敢學習
   - 退化為純物理模型

3. **Waveform L2 不等於 Phase 正確**
   - L2 主要由 magnitude 主導
   - Phase 誤差被 amplitude 加權
   - 低能量頻段的 phase 被忽略

### 對比所有方法

| 方法 | L2 | IPD | Phase | 結論 |
|------|-----|-----|-------|------|
| DPATFNet | 0.000719 | 3.07 | 3.18 | Phase 隨機 |
| HybridTFNet-IPD | 0.000549 | 3.07 | 3.18 | Phase 隨機 |
| WaveformSpatializer | 0.000634 | 0.8773 | 0.9967 | Phase 仍差 |
| **HybridPhysical** | **0.000830** | **0.9148** | **0.9992** | **Phase 仍差** |

**結論**：
- L2 沒有突破 DPATFNet
- IPD 有改善但不夠
- Phase 完全沒有改善

### 為什麼所有方法都失敗？

#### 核心問題：資訊不足

```
I(Phase; Mono, View) ≈ 0

無論用什麼方法：
- STFT-based
- Waveform-based  
- Physical + Learning

都無法從 mono 預測 Phase
```

#### 物理模型的局限

```python
# 物理 ITD 只是簡化模型
ITD = (r/c) × (sin(θ) + θ)

實際 HRTF:
- 頭部不是完美球體
- 耳廓、肩膀的影響
- 頻率相關的複雜變化
- 個體差異

→ 物理模型不夠準確
→ 但從 mono 也學不到修正
```

### 可能的改進（但可能仍然無效）

#### 選項 1：增加 Phase loss 權重

```python
loss = 1.0 × Waveform_L2
     + 0.5 × Magnitude
     + 1.0 × IPD
     + 10.0 × Phase_L    # 新增
     + 10.0 × Phase_R    # 新增
     + 0.001 × ITD_reg   # 減小正則化
```

**問題**：
- 之前試過，Phase 仍然學不會
- 可能導致訓練不穩定

#### 選項 2：放寬 ITD 正則化

```python
lambda_itd_reg: 0.01 → 0.0001
```

**問題**：
- 可能學到錯誤的修正
- 沒有足夠的監督信號

#### 選項 3：頻率自適應策略

```python
# 低頻: 純物理 ITD
# 中頻: 物理 + 學習
# 高頻: 放棄 Phase，只學 ILD
```

**問題**：
- 更複雜
- 仍然無法解決高頻 Phase

### 殘酷的事實

**經過 7 個實驗，所有方法都證明了同一件事**：

```
從 mono 音訊無法學習雙耳 Phase

這不是：
- 架構問題
- Loss 問題
- 訓練策略問題

這是：
- 資訊理論限制
- 物理不可能
```
