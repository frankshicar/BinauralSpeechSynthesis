### 3. 噪音問題調查與修復 ✅

#### 根本原因

經過詳細對比分析，發現噪音的主要原因是 [evaluate.py](file:///home/sbplab/frank/BinauralSpeechSynthesis/evaluate.py) 中 [chunked_forwarding](file:///home/sbplab/frank/BinauralSpeechSynthesis/evaluate.py#61-136) 函數的錯誤實作：

**問題 1: 缺少 clamp 操作**
```python
# 原始版本（無噪音）
binaural = th.clamp(binaural, min=-1, max=1).cpu()

# 錯誤版本（有噪音）
binaural = th.cat(binaural_segments, dim=-1)
return binaural  # 沒有 clamp！
```

**問題 2: 錯誤的 Context Padding 實作**
- 使用 1 秒小片段（48000 samples）而非 10 秒（480000 samples）
- 增加了額外的 2400 samples padding，但計算不正確
- 每個片段的邊界處理有誤差累積

#### 修復方案

**採用方案 A**: 恢復原始實作並加上註解

```python
def chunked_forwarding(net, mono, view):
    """
    分段處理雙耳化 (Binauralize mono input in chunks)
    使用 Overlap-Discard 策略處理長音訊，避免記憶體溢出
    """
    # 使用 10 秒片段
    chunk_size = 480000
    rec_field = net.receptive_field() + 1000
    rec_field -= rec_field % 400
    
    # ... 處理邏輯 ...
    
    # 關鍵修復：限制範圍到 [-1, 1]
    binaural = th.clamp(binaural, min=-1, max=1).cpu()
    return binaural
```

**預期效果**: 噪音應該完全消失
