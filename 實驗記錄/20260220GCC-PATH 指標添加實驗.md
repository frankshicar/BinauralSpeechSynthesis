# 實施計畫：加入角度誤差指標 (MAAE)

## 背景與目標 (Background & Goal)

目前模型雖然有 ITD (時間差) 和 ILD (強度差) 指標，但這些是「間接」的空間定位指標，無法直接告訴我們：
- **模型輸出的角度是否正確**  
- **與真實角度差幾度**

### 提議方案 (Proposed Solution)

加入 **Mean Absolute Angular Error (MAAE)**，使用 **GCC-PHAT** 作為「第三方裁判」：

1. **Ground Truth Angle (θ_gt)**: 從 testset metadata 讀取（Subject 1~7 對應 -90°, -60°, -30°, 0°, +30°, +60°, +90°）
2. **Predicted Angle (θ_pred)**: 使用 GCC-PHAT 算法從模型輸出雙耳音訊中估計角度
3. **MAAE 公式**: `MAAE = (1/N) * Σ|θ_pred - θ_gt|`

### 文獻支持
- Varzandeh et al., 2024
- Kim et al. 系列

---

## User Review Required

> [!IMPORTANT]
> **GCC-PHAT 的角度估計精度依賴於 HRTF 假設**  
> 由於 testset 使用的是我們自己收錄的 mannequin head，而非標準 KEMAR，GCC-PHAT 可能會有系統性誤差。但這不影響「相對比較」的價值，因為所有模型都使用相同的裁判。

## Proposed Changes

### Component 1: DOA Estimation Module

#### [NEW] [doa.py](file:///home/sbplab/frank/BinauralSpeechSynthesis/src/doa.py)

**新增 Direction of Arrival (DOA) 估計模組**，包含：

1. **GCC-PHAT 實作** (`gcc_phat_estimate`)
   - 輸入：雙耳音訊 (2 x T)
   - 輸出：估計的方位角 (degrees)
   - 算法：
     ```
     1. 計算左右聲道的互功率譜 (Cross-Power Spectrum)
     2. 相位變換 (Phase Transform): R(ω) = X_L(ω) * conj(X_R(ω)) / |X_L(ω) * X_R(ω)|
     3. 反傅立葉變換得到互相關函數
     4. 找到峰值對應的時間延遲 τ
     5. 轉換為角度: θ = arcsin(c * τ / d)，其中 c=343m/s，d=耳間距≈0.215m
     ```

2. **輔助函數**
   - `itd_to_azimuth`: 將 ITD (微秒) 轉換為方位角 (度)
   - `validate_angle`: 驗證估計角度是否在合理範圍 [-90°, +90°]

3. **單元測試** (可選)
   - 使用已知角度的合成音訊驗證算法正確性

**新增 `AngularErrorLoss` 類別**：

```python
class AngularErrorLoss(Loss):
    def __init__(self, ground_truth_angle, sample_rate=48000, mask_beginning=0):
        '''
        計算預測雙耳音訊與目標角度之間的角度誤差
        
        :param ground_truth_angle: (float) 目標方位角（度）
        :param sample_rate: (int) 音訊採樣率
        :param mask_beginning: (int) 開頭遮罩樣本數
        '''
        super().__init__(mask_beginning)
        self.ground_truth_angle = ground_truth_angle
        self.sample_rate = sample_rate
    
    def _loss(self, data, target):
        '''
        :param data: 預測的雙耳訊號 (B x 2 x T)
        :param target: 參考訊號（此處不使用，僅為符合接口）
        :return: 角度誤差（度）
        '''
        from src.doa import gcc_phat_estimate
        
        batch_errors = []
        for b in range(data.shape[0]):
            pred_angle = gcc_phat_estimate(
                data[b].cpu().numpy(), 
                sample_rate=self.sample_rate
            )
            error = abs(pred_angle - self.ground_truth_angle)
            batch_errors.append(error)
        
        return th.tensor(batch_errors).mean()
```

## 實施注意事項 (Implementation Notes)

> [!TIP]
> **GCC-PHAT 參數調整**  
> 如果初步結果顯示估計角度不穩定，可以調整：
> - STFT 窗口大小（預設 2048）
> - 頻率範圍限制（僅使用 500Hz-8kHz）
> - 平滑化處理（多幀平均）

> [!WARNING]
> **角度歧義性**  
> GCC-PHAT 無法區分前方/後方（front-back confusion），因為我們的 testset 都在前半球，這不是問題。但如果未來擴展到 360°，需要使用更進階的方法。
