## 1. 核心問題確認

### 1.1 MAA/MAMA 標準與當前模型差距

人耳對聲源方位的感知精度由兩個心理聲學指標衡量：

- **MAA（Minimum Audible Angle）**：靜態聲源的最小可辨角度差。Chandler & Grantham（1992）測量水平面正前方 MAA 約為 **1-3°**，Mills（1958）的經典測量在 0° 方位角時為 **~1°**。
- **MAMA（Minimum Audible Movement Angle）**：移動聲源的最小可辨移動角度。Chandler & Grantham（1992）測量在 10°/s 速度下 MAMA 約為 **8.8°**。

v5b 的表現對比：

| 指標 | 人耳標準 | v5b 表現 | 差距倍數 |
|------|---------|---------|---------|
| MAA | ~1.2° | ~19.1° | **15.8×** |
| MAMA (10°/s) | ~8.8° | ~19.1° | **2.2×** |
| Meta 3-block baseline | — | 4.0° | — |

v5b 不僅遠超 MAA，甚至超過 MAMA 兩倍以上，意味著即使是移動聲源的感知也無法達標。感知上，19° 的誤差使聲源位置完全模糊，聽者無法判斷聲音來自正前方還是側方 20° 處。

`val_ipd = 2.2649 rad` 是另一個關鍵指標。理想情況下，若模型完整保留 GeometricWarper 的 ITD，低頻 IPD 應接近 0（phase residual 不應改變雙耳相位差）。2.26 rad 的 IPD 誤差表明低頻相位結構完全未被正確學習，這與 Wightman & Kistler（1992）的研究一致——低頻（< 1500 Hz）ITD 是水平面方位感知的主要線索，此頻段的相位誤差直接導致方位感知失敗。

### 1.2 warpfield 的物理意義與現有問題

`GeometricTimeWarper.displacements2warpfield` 計算的 warpfield 代表聲音從聲源到每隻耳朵的傳播延遲（以 samples 為單位）：

```python
distance = th.sqrt(th.sum(displacements**2, dim=2) + 1e-8)  # B x 2 x K
distance = F.interpolate(distance, size=seq_length)          # B x 2 x T
warpfield = -distance / 343.0 * self.sampling_rate           # B x 2 x T
```

物理意義：`warpfield[b, 0, t]` 為左耳在時刻 t 的傳播延遲（samples），`warpfield[b, 1, t]` 為右耳延遲。雙耳時間差 ITD = `warpfield_left - warpfield_right`。

**現有問題：ear_offset 低估**

```python
left_ear_offset  = [0, -0.08, -0.22]   # y = -0.08m
right_ear_offset = [0,  0.08, -0.22]   # y = +0.08m
```

耳間距 = 2 × 0.08m = **0.16m**，但 KEMAR 假人標準耳間距為 **0.215m**（GCC-PHAT 評估時使用此值）。

數值影響（以 ±90° 側向聲源為例）：
- 現有設定最大 ITD：0.16 / 343 × 48000 ≈ **22.4 samples**（467 μs）
- KEMAR 標準最大 ITD：0.215 / 343 × 48000 ≈ **30.1 samples**（627 μs）
- 低估量：**7.7 samples**，對應 **160 μs**

Klumpp & Eady（1956）測量 ITD JND（Just Noticeable Difference）為 **10-20 μs**。160 μs 的系統性誤差是 JND 的 8-16 倍，在低頻完全可被人耳察覺。即使模型完美學習了 warpfield 所定義的 ITD，與 GT（基於 KEMAR 0.215m 生成）之間仍存在無法消除的系統性偏差。

此外，現有 warpfield 缺少 **Woodworth 球形頭部繞射修正**（Woodworth, 1954）。Woodworth 模型考慮聲波繞過球形頭部的路徑延長效應：

```
ITD_woodworth(θ) = (r/c) × (sin θ + θ)   # r = 頭部半徑, θ = 方位角
```

Kuhn（1977）進一步指出低頻（λ >> 頭部尺寸）和高頻的 ITD 有所不同，低頻 ITD 約為高頻的 1.5 倍。忽略此修正在側向（±90°）會造成額外的 ITD 低估。

### 1.3 ITD 被 phase residual 抹掉的機制

這是 v5b 最關鍵的失效機制，可用博弈論框架解釋（Sener & Koltun, 2018）。

GeometricWarper 在時域施加 ITD 後，訊號進入 STFT 域。FiLM ResStack 學習的是**相位殘差**（phase residual）。Phase loss 的最優解不唯一：

設左耳相位殘差為 `φ_L(f,t)`，右耳為 `φ_R(f,t)`。若模型學到：
```
φ_L(f,t) = δ(f,t)
φ_R(f,t) = δ(f,t)   # 相同的偏移
```

則 Phase loss ≈ 0（兩耳相位都對齊了 GT），但 IPD = `φ_L - φ_R = 0`，GeometricWarper 建立的 ITD 被完全抹除。

根據 Keskar et al.（2017）的研究，SGD 傾向於收斂到損失曲面的「寬廣低谷」（flat minima）。「兩耳加相同偏移」這個解對應一個非常寬廣的流形（任意 δ 都滿足），因此 SGD 天然傾向於找到這個解。

v5b 嘗試用 `ITDConsistencyLoss` 對抗此問題，但失效原因在於：Stage 1 初期模型的 phase residual ≈ 0，loss = 0，無梯度信號，模型在有梯度之前已收斂到錯誤的吸引子。

## 2. 聲學專家分析

### 2.1 雙耳聽覺線索的頻率分工

根據 Wightman & Kistler（1992）的奠基性研究，水平面方位感知的線索有明確的頻率分工：

- **低頻（< 1500 Hz）**：ITD（Interaural Time Difference）主導。此頻段波長長，相位在雙耳間保持一致性，ITD 可被精確感知。
- **高頻（> 1500 Hz）**：ILD（Interaural Level Difference）和 pinnae（耳廓）頻譜線索主導。此頻段波長短，相位模糊，但耳廓對不同方向的聲音產生特徵性的頻譜凹陷（notch），提供仰角和前後辨別線索。

v5b 的 `val_ipd = 2.26 rad` 直接反映低頻 ITD 線索的失敗。正確的低頻 IPD 應接近 0（phase residual 不應改變雙耳相位差），2.26 rad 意味著模型在低頻引入了大量錯誤的雙耳相位差。

### 2.2 ear_offset 的聲學合理性分析

KEMAR（Knowles Electronics Manikin for Acoustic Research）是雙耳音訊研究的標準假人，其耳間距為 **0.215m**（Shaw, 1974）。訓練資料的 GT 若基於 KEMAR 或真實人頭錄音生成，其 ITD 對應的有效耳間距約為 0.215m。

現有 `ear_offset y = ±0.08m`（耳間距 0.16m）的來源可能是簡化的幾何模型，未考慮：
1. 頭部不是點，聲波需繞過球形頭部（Woodworth, 1954）
2. 耳廓位置在頭部側面，有效聲學中心並非幾何中心

**修正建議**：將 `ear_offset y` 從 ±0.08m 修正為 ±0.1075m，使耳間距從 0.16m 增加到 0.215m，與 KEMAR 標準對齊。

**Woodworth 修正**（Woodworth, 1954；Kuhn, 1977）：

對於球形頭部模型，ITD 的精確計算為：
```
ITD(θ) = (a/c) × (sin θ + θ)
```
其中 `a` 為頭部半徑（~0.0875m），`θ` 為方位角（弧度）。此修正在 θ = 90° 時使 ITD 增加約 57%（相比純幾何距離差）。Kuhn（1977）進一步指出低頻 ITD 比高頻大約 1.5 倍，這是由於低頻聲波繞射路徑更長。

### 2.3 FiLM 注入 IPD 的聲學可行性

FiLM（Feature-wise Linear Modulation，Perez et al., 2018）的頻帶調製機制與聽覺濾波器組（ERB，Equivalent Rectangular Bandwidth）天然對齊。ERB 濾波器組在低頻密集、高頻稀疏，與 STFT 的線性頻率分辨率不同，但 FiLM 的 64-band 設計可以近似此分布。

直接注入 IPD 比注入 position encoding 更有效的聲學理由：
- Position encoding 是空間座標（x, y, z），模型需要學習從座標到 ITD 的映射，這是一個非線性幾何計算
- 直接注入 ITD/IPD 提供的是**感知直接線索**，模型只需學習如何利用此線索調整頻譜，而非重新計算幾何

分頻注入的聲學依據：
- 低頻（< 1500 Hz）：注入 IPD（cos, sin 雙通道表示，避免相位不連續）
- 高頻（> 1500 Hz）：注入 ILD（dB 尺度，對應人耳的對數感知）

使用 (cos φ, sin φ) 表示 IPD 的理由：相位是週期量，直接用數值表示在 ±π 邊界不連續，cos/sin 表示保持連續性，與 Transformer 的 positional encoding 設計原理相同（Vaswani et al., 2017）。

## 3. 深度學習專家分析

### 3.1 Phase Residual 抹掉 ITD 的梯度動力學

從優化理論角度，Phase loss 的損失曲面存在一個高維流形，其上所有點的 loss 值相同但 IPD 為零。這是多任務學習中的梯度競爭問題（Sener & Koltun, 2018）。

形式化描述：設 Phase loss 為：
```
L_phase = ||angle(Y_pred) - angle(Y_gt)||²
```

最優解集合為：
```
S* = {(φ_L, φ_R) : φ_L = angle(Y_gt_L) + δ, φ_R = angle(Y_gt_R) + δ, ∀δ}
```

當 `δ_L = δ_R = δ` 時，`L_phase = 0` 且 `IPD_pred = φ_L - φ_R = IPD_gt_L - IPD_gt_R`，但若 GeometricWarper 的 ITD 被 phase residual 的 δ 抵消，則實際輸出的 IPD 為零。

根據 Keskar et al.（2017），SGD 傾向於收斂到 Hessian 最大特徵值小的「平坦」解，而「兩耳加相同偏移」的解對應一個非常平坦的流形，因此是 SGD 的自然吸引子。

### 3.2 ITDConsistencyLoss 失效的根本原因

```python
# 假設的 ITDConsistencyLoss 實現
loss_itd = ||IPD_pred - IPD_geo||²
```

失效機制：Stage 1 初期，模型輸出接近零（隨機初始化），phase residual ≈ 0，因此 `IPD_pred ≈ IPD_geo`，`loss_itd ≈ 0`，梯度為零。模型在 Phase loss 的引導下先收斂到「兩耳加相同偏移」的吸引子，之後 ITDConsistencyLoss 即使有梯度也難以逃脫此局部最優。

這是課程學習（curriculum learning）設計失敗的典型案例（Bengio et al., 2009）：應該在訓練初期就施加 ITD 約束，而非等到 Phase loss 已收斂。

### 3.3 IPD Loss 的三個設計缺陷

v5b 的 IPD loss 存在以下問題：

**缺陷 1：全頻段平均**  
高頻（> 1500 Hz）的 IPD 本身就是模糊的（相位纏繞），強制模型在高頻學習 IPD 會引入噪聲梯度，稀釋低頻的有效梯度。

**缺陷 2：MSE 對 outlier 敏感**  
相位是週期量，MSE 在相位跳變（如 π → -π）時會產生極大的梯度，導致訓練不穩定。

**缺陷 3：能量 mask 設計不當**  
若 mask 未正確排除低能量幀，靜音段的相位噪聲會污染梯度。

**改進方案**：
```python
# 只用低頻 bins（< 1500 Hz）
low_freq_bins = n_fft // 2 * 1500 // (sample_rate // 2)  # ~32 bins @ n_fft=1024, sr=48000

# Cosine distance（對相位週期性友好）
ipd_pred = phase_L_pred - phase_R_pred  # [B, F, T]
ipd_gt   = phase_L_gt   - phase_R_gt

loss_ipd = 1 - torch.mean(
    torch.cos(ipd_pred[:, :low_freq_bins, :] - ipd_gt[:, :low_freq_bins, :])
    * energy_mask[:, :low_freq_bins, :]
)
```

### 3.4 FiLM 注入 Geometric ITD 的架構設計

最有效的方案是直接注入 GeometricWarper 計算的 ITD 時間序列（選項 A）：

```python
class ITDConditioner(nn.Module):
    def __init__(self, itd_dim=32):
        super().__init__()
        # 將 ITD 時間序列壓縮為固定維度向量
        self.encoder = nn.Sequential(
            nn.Conv1d(1, 16, kernel_size=7, padding=3),
            nn.ReLU(),
            nn.Conv1d(16, itd_dim, kernel_size=7, padding=3),
            nn.AdaptiveAvgPool1d(1)  # 時間維度壓縮
        )
    
    def forward(self, warpfield):
        # warpfield: [B, 2, T]
        itd = warpfield[:, 0:1, :] - warpfield[:, 1:2, :]  # [B, 1, T]
        return self.encoder(itd).squeeze(-1)  # [B, itd_dim]

# 在 FiLM conditioning 中：
# 原始：condition = fourier_pos_enc(position)  # [B, 256]
# 修改：condition = concat([fourier_pos_enc(position), itd_conditioner(warpfield)], dim=-1)  # [B, 288]
```

此設計的優點：
- ITD 是純幾何量，不依賴梯度，前向傳播即可修正
- 32D 向量足以表示 ITD 的時間變化模式（Bengio et al., 2013 的表示學習原則）
- 與現有 FiLM 架構相容，只需修改 conditioning 維度

## 4. 關鍵問題解答

### 4.1 warpfield 是否需要修正？

**結論：ear_offset 不應修改；應加入 neural warpfield 修正機制。**

> **【修正】** 原報告建議將 ear_offset 從 0.08m 改為 0.1075m 是錯誤的。確認 Meta 原始 code（`backup_original/src/models.py`）的 ear_offset 就是 [0, ±0.08, -0.22]，這是 tracking marker 到耳朵的實際幾何量測值，不是近似值。修改此值會讓 GeometricWarper 的幾何計算與真實錄音不符，Phase/L2 loss 都會變差。

**正確理解**：Meta 的 `Warpnet` 設計如下：

```python
# Meta Warpnet.forward()
geometric_warpfield = self.geometric_warper._warpfield(view, seq_length)
neural_warpfield    = self.neural_warpfield(view, seq_length)   # 4層 Conv1d 學習修正量
warpfield           = geometric_warpfield + neural_warpfield    # 合併
warpfield           = -F.relu(-warpfield)                       # causality
warped = self.neural_warper(mono_stereo, warpfield)
```

`neural_warpfield` 是一個輕量的 4 層 Conv1d 網路，學習修正 geometric warpfield 低估的 ITD。最終施加到音訊的是 `geometric + neural` 的合併 warpfield，而非純幾何值。

**v5b 的問題**：直接用 `geometric_warper(mono, view)` 做時域 warping，沒有 neural 修正步驟。FiLM ResStack 在 STFT 域操作，無法修正時域 ITD 的低估。

**v6 修正方案**：加入 `NeuralWarpCorrector`（仿照 Meta Warpnet），在 GeometricWarper 之後、STFT 之前修正 warpfield：

```python
class NeuralWarpCorrector(nn.Module):
    def __init__(self, view_dim=7, channels=64, layers=4):
        super().__init__()
        self.layers = nn.ModuleList([
            nn.Conv1d(view_dim if l == 0 else channels, channels, kernel_size=2)
            for l in range(layers)
        ])
        self.linear = nn.Conv1d(channels, 2, kernel_size=1)
        self.neural_warper = MonotoneTimeWarper()

    def forward(self, mono, view, geometric_warpfield):
        x = view
        for layer in self.layers:
            x = F.pad(x, [1, 0])
            x = F.relu(layer(x))
        delta = F.interpolate(self.linear(x), size=mono.shape[-1])
        warpfield = -F.relu(-(geometric_warpfield + delta))  # causality
        return self.neural_warper(th.cat([mono, mono], dim=1), warpfield)
```

在 `GeoWarpFiLMNet.forward()` 中：
```python
# v5b
y_init = self.geometric_warper(mono, view)

# v6
geo_wf = self.geometric_warper._warpfield(view, mono.shape[-1])
y_init = self.neural_warp_corrector(mono, view, geo_wf)  # geometric + neural
```

### 4.2 FiLM 注入 IPD 的可行性與設計方案

**結論：可行，且優於現有 position encoding 方案。**

**可行性依據**：
- FiLM 的 γ(c) 和 β(c) 是對特徵圖的逐頻帶縮放和偏移，天然適合注入頻率相關的雙耳線索
- 現有 position encoding 需要模型學習「座標 → ITD」的非線性映射；直接注入 ITD 省去此步驟
- Perez et al.（2018）的 FiLM 原始論文已證明此機制在條件生成任務中的有效性

**設計方案（三層遞進）**：

**方案 A（最高優先，P0）**：注入 Geometric ITD
```python
# ITDConditioner: warpfield [B,2,T] → itd_vec [B,32]
itd_vec = ITDConditioner()(warpfield)
condition = torch.cat([pos_enc, itd_vec], dim=-1)  # 256+32=288D
```

**方案 B（P1）**：分頻注入 IPD + ILD
```python
# 從 GT STFT 計算
ipd_low = phase_L[:, :32, :] - phase_R[:, :32, :]   # 低頻 IPD
ild_high = mag_L[:, 32:, :].log() - mag_R[:, 32:, :].log()  # 高頻 ILD（dB）

# 編碼為 conditioning vector
ipd_enc = torch.cat([ipd_low.cos(), ipd_low.sin()], dim=1)  # 避免相位不連續
```

### 4.3 Meta 已避免 ITD 被繞過但角度誤差仍大的原因

Meta 的 GeometricWarper 在**時域**施加 ITD，理論上 FiLM ResStack 無法在時域「撤銷」此延遲（因為 ResStack 在 STFT 域操作）。然而角度誤差仍大，原因是多層次的：

**原因 1：warpfield 本身的幾何誤差（最根本）**  
如 4.1 所述，ear_offset 低估導致 warpfield 定義的 ITD 就是錯的。GeometricWarper 「避免繞過」的是一個錯誤的 ITD 值。

**原因 2：STFT 域的 phase residual 可以改變 IPD**  
雖然時域 ITD 被保留，但 FiLM ResStack 在 STFT 域修改相位後，輸出訊號的 IPD（頻域雙耳相位差）可以與時域 ITD 不一致。GCC-PHAT 評估的是頻域 IPD，而非時域 ITD，因此 phase residual 仍然影響評估結果。

**原因 3：FiLM 條件信號不足**  
Position encoding 提供的是空間座標，模型需要自行推斷應該保留多少 ITD。缺乏明確的 ITD 約束，模型傾向於優化 Phase loss 而犧牲 IPD 一致性。

**原因 4：Loss 設計未強制低頻 IPD 保留**  
全頻段 IPD loss 被高頻噪聲稀釋，低頻梯度不足以對抗 Phase loss 的吸引子效應。

## 5. v6 架構提案

### 5.1 Neural Warpfield 修正（取代原 ear_offset 修正方案）

> **【修正】** 原方案「修改 ear_offset」已廢棄。正確方案是加入 `NeuralWarpCorrector`，仿照 Meta Warpnet 的 `geometric + neural` 設計。

**完整 v6 前向流程**：

```
mono + view
  ↓
GeometricWarper._warpfield()     → geometric_warpfield (B, 2, T)
  ↓
NeuralWarpCorrector(view)        → delta (B, 2, T)  [學習 ITD 修正量]
  ↓
warpfield = geometric + delta    → MonotoneTimeWarper → y_init (B, 2, T)
  ↓

  ↓

  ↓
```

**NeuralWarpCorrector 參數量**：4 層 Conv1d(64) ≈ 33K 參數，相比整體模型（~2M）可忽略。

### 5.2 FiLM IPD 注入設計

**完整的 v6 Conditioning 模組**：

```python
class V6Conditioner(nn.Module):
    """
    v6 FiLM conditioning：Fourier pos enc + Geometric ITD
    """
    def __init__(self, pos_enc_dim=256, itd_enc_dim=32):
        super().__init__()
        self.pos_enc_dim = pos_enc_dim
        # ITD 時間序列編碼器
        self.itd_encoder = nn.Sequential(
            nn.Conv1d(1, 16, kernel_size=7, padding=3),
            nn.GELU(),
            nn.Conv1d(16, itd_enc_dim, kernel_size=7, padding=3),
            nn.AdaptiveAvgPool1d(1),
            nn.Flatten()
        )
        self.out_dim = pos_enc_dim + itd_enc_dim  # 288
    
    def forward(self, position, warpfield):
        # position: [B, 3] 空間座標
        # warpfield: [B, 2, T] 左右耳傳播延遲
        
        pos_enc = fourier_position_encoding(position, self.pos_enc_dim)  # [B, 256]
        
        itd = warpfield[:, 0:1, :] - warpfield[:, 1:2, :]  # [B, 1, T]
        itd_enc = self.itd_encoder(itd)  # [B, 32]
        
        return torch.cat([pos_enc, itd_enc], dim=-1)  # [B, 288]
```

### 5.3 Loss 函數調整

**v6 Loss 設計**：

```python
class V6Loss(nn.Module):
    def __init__(self, sample_rate=48000, n_fft=1024):
        super().__init__()
        # 低頻截止 bin（< 1500 Hz）
        self.low_freq_bins = int(n_fft // 2 * 1500 / (sample_rate / 2))  # ~32
    
    def forward(self, pred, gt, warpfield):
        # 1. L2 loss（保持不變）
        loss_l2 = F.mse_loss(pred, gt)
        
        # 2. Phase loss（保持不變）
        phase_pred = torch.angle(torch.stft(pred, ...))
        phase_gt   = torch.angle(torch.stft(gt, ...))
        loss_phase = F.mse_loss(phase_pred, phase_gt)
        
        # 3. 低頻 IPD loss（cosine distance，僅低頻 bins）
        ipd_pred = phase_pred[:, 0, :self.low_freq_bins, :] \
                 - phase_pred[:, 1, :self.low_freq_bins, :]
        ipd_gt   = phase_gt[:, 0, :self.low_freq_bins, :] \
                 - phase_gt[:, 1, :self.low_freq_bins, :]
        
        energy_mask = (torch.abs(torch.stft(gt, ...))[:, :, :self.low_freq_bins, :] > 1e-4)
        loss_ipd = 1 - (torch.cos(ipd_pred - ipd_gt) * energy_mask).sum() \
                       / (energy_mask.sum() + 1e-8)
        
        # 4. IPD Preservation loss（懲罰 phase residual 在低頻改變 IPD）
        # 確保 phase_L_res - phase_R_res ≈ 0 在低頻
        phase_res_L = phase_pred[:, 0, :self.low_freq_bins, :] \
                    - warpfield_to_phase(warpfield[:, 0, :], ...)
        phase_res_R = phase_pred[:, 1, :self.low_freq_bins, :] \
                    - warpfield_to_phase(warpfield[:, 1, :], ...)
        loss_ipd_preserve = torch.mean((phase_res_L - phase_res_R) ** 2)
        
        return (loss_l2 
                + 1.0 * loss_phase 
                + 2.0 * loss_ipd           # 提高低頻 IPD 權重
                + 0.5 * loss_ipd_preserve)  # 防止 phase residual 抹掉 ITD
```

**關鍵改動**：
- `loss_ipd`：從全頻段 MSE → 低頻 cosine distance（前 32 bins）
- `loss_ipd_preserve`：新增，直接懲罰 phase residual 在低頻改變 IPD 的行為
- 權重：`ipd_weight = 2.0`（提高，因為低頻 IPD 是方位感知的主要線索）
