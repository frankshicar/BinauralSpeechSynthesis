# BinauralSpeechSynthesis v6 架構分析報告

**日期**：2026-05-01  
**版本**：v6 設計報告  
**基礎**：v5b 實驗結果分析 + 深度學習架構專家討論

---

## 摘要

v5b 達到 val_phase=0.853，但 angular error 仍未改善（預估仍在 15-20° 範圍）。
本報告從深度學習架構角度分析根本原因，並提出 v6 具體修改方案。

核心結論：
1. Phase residual 抹掉 ITD 是**梯度競爭下的 Nash 均衡解**，不是 bug
2. warpfield 的 ear_offset 誤差（0.08m vs 理論 0.1075m）是**系統性偏差**，必須修正
3. FiLM 注入 geometric ITD（選項 A）是最有效的架構改進方向
4. IPD loss 的設計有根本缺陷：數值尺度與 Phase loss 不匹配

## 1. 為什麼 Phase Residual 會抹掉 ITD

### 1.1 問題重述

架構流程：`GeometricWarper → STFT → FiLM ResStack → 輸出`

從 v5 實驗記錄可以確認：
```
subject1  gt=-90°  GeoWarp_ITD=+23smp  Model_ITD=+0smp  → 角度誤差 66.5°
subject7  gt=+90°  GeoWarp_ITD=-23smp  Model_ITD=+0smp  → 角度誤差 0.0°（符號相反）
```

模型選擇了「每隻耳朵的絕對相位接近 GT，但 ITD=0」的解。

### 1.2 梯度競爭分析：Nash 均衡解

這是一個典型的**多目標優化 Nash 均衡**問題（Sener & Koltun, 2018）。

設 Phase loss 為：
```
L_phase = E[|∠Y_L_pred - ∠Y_L_gt| + |∠Y_R_pred - ∠Y_R_gt|]
```

設 IPD loss 為：
```
L_ipd = E[|(∠Y_L_pred - ∠Y_R_pred) - (∠Y_L_gt - ∠Y_R_gt)|]
```

**關鍵觀察**：Phase loss 的梯度方向是讓每隻耳朵的絕對相位分別接近 GT。
對於一個 ITD=τ 的信號，GT 的相位關係是：
```
∠Y_L_gt(f) - ∠Y_R_gt(f) = 2πfτ/fs  （線性相位斜率）
```

Phase loss 的最優解**不唯一**：只要 ∠Y_L_pred ≈ ∠Y_L_gt 且 ∠Y_R_pred ≈ ∠Y_R_gt，
loss 就最小。但這個條件**不要求** ∠Y_L_pred - ∠Y_R_pred = 2πfτ/fs。

具體來說，模型可以找到一個解：
```
∠Y_L_pred = ∠Y_L_gt + δ(f,t)
∠Y_R_pred = ∠Y_R_gt + δ(f,t)   ← 同一個 δ
```

此時 Phase loss ≈ 0（兩耳都接近 GT），但 IPD = 0（因為 δ 相同，相位差被消除）。

這個解在 Phase loss 的 loss landscape 上是一個**寬廣的低谷**（flat minimum），
而 ITD≠0 的解是一個**窄峰**（sharp minimum）。
根據 Keskar et al. (2017) 的 sharp/flat minima 理論，SGD 傾向於收斂到 flat minimum，
因此模型自然選擇了 ITD=0 的解。

### 1.3 為什麼 ITDConsistencyLoss 無效

v5 的實驗記錄顯示 `itd=0.0000`（從 Epoch 2 開始）。

原因：`ITDConsistencyLoss` 計算的是 model cross-spectrum phase 與 GeoWarp cross-spectrum phase 的差異。
但在 Stage 1 初期，model 的 phase residual ≈ 0（還沒學到任何修正），
所以 model 的 cross-spectrum phase ≈ GeoWarp 的 cross-spectrum phase，loss = 0，無梯度。

更根本的問題：即使 ITDConsistencyLoss 有效，它只能讓 model 的 ITD 等於 GeoWarp 的 ITD，
但 GeoWarp 的 ITD 本身就低估了（23 samples vs 需要的 ~30 samples），
所以角度誤差仍然存在。

### 1.4 為什麼 IPD weight=1.0 仍然無效

v5b 強制 `w_ipd = w_phase = 1.0`，但 val_ipd 仍在 2.2-2.4。

問題在於 IPD loss 的**數值尺度**：
- Phase loss 的值域：[0, π]，典型值 ~0.85 rad
- IPD loss 的值域：[0, π]，典型值 ~2.2 rad

兩者數值尺度相近，但 IPD loss 的**梯度方向**與 Phase loss 衝突：
- Phase loss 梯度：讓每耳絕對相位接近 GT
- IPD loss 梯度：讓左右耳相位差接近 GT

當 Phase loss 已經很小（0.85 rad），繼續降低它的梯度方向
與降低 IPD loss 的梯度方向**不平行**，甚至**相反**。

這是 Gradient Interference 問題（Yu et al., 2020, PCGrad）：
兩個 loss 的梯度夾角 > 90°，相加後反而互相抵消，
導致兩個 loss 都無法有效下降。

## 2. Warpfield 修正分析

### 2.1 現有 warpfield 的系統性誤差

GeometricWarper 的 ear offset：
```python
left_ear_offset  = [0, -0.08, -0.22]  # m
right_ear_offset = [0,  0.08, -0.22]  # m
```

耳間距 = 2 × 0.08 = **0.16m**

但 GCC-PHAT 使用 `ear_distance=0.215m`（KEMAR 標準值）。

在 ±90° 時，ITD 的理論值：
```
ITD_geo    = 0.16 / 343.0 = 0.000467s = 22.4 samples @ 48kHz
ITD_kemar  = 0.215 / 343.0 = 0.000627s = 30.1 samples @ 48kHz
```

**誤差**：22.4 vs 30.1 samples，差距 25.4%。

這個誤差導致 GCC-PHAT 在 ±90° 時估出的角度只有 ±53.5°（而非 ±90°），
即使 model 完全保留了 GeoWarp 的 ITD，角度誤差仍有 36.5°。

### 2.2 ear_offset 的正確值

KEMAR 假人頭的耳間距（ear-to-ear distance）= 0.215m，
因此每側耳朵到頭部中心的距離 = 0.1075m。

正確的 ear offset 應為：
```python
left_ear_offset  = [0, -0.1075, -0.22]  # 修正 y 分量
right_ear_offset = [0,  0.1075, -0.22]
```

修正後，±90° 的 ITD = 0.215/343.0 × 48000 = **30.1 samples**，
與 GCC-PHAT 的 ear_distance=0.215m 完全一致。

### 2.3 修正 warpfield 的梯度流動分析

warpfield 的計算路徑：
```
view → _3d_displacements → displacements2warpfield → TimeWarper → y_init → STFT → FiLM → output
```

**關鍵**：`displacements2warpfield` 中的 `distance` 計算：
```python
distance = th.sqrt(th.sum(displacements**2, dim=2) + 1e-8)
warpfield = -distance / 343.0 * self.sampling_rate
```

ear_offset 是一個**常數張量**（不是可學習參數），
修正它不需要任何梯度計算，只是改變了 warpfield 的初始值。

**修正 warpfield 的效果**：
- 直接改變 `y_init` 的 ITD（從 22.4 samples 到 30.1 samples）
- 這個改變在 STFT 之前發生，因此 FiLM ResStack 看到的輸入特徵就已經有正確的 ITD
- 模型不需要「學習」如何修正 ITD，只需要「保留」正確的 ITD

**對比 loss 修正的梯度流動**：
- Loss 修正需要梯度從 output 反傳到 phase residual head
- 但 phase residual head 的梯度方向由 Phase loss 主導（因為 Phase loss 數值更穩定）
- IPD loss 的梯度在 Phase loss 的強大梯度面前被淹沒

**結論**：修正 warpfield 是**前向傳播的修正**，不依賴梯度；
修正 loss 是**反向傳播的修正**，受梯度競爭影響。
前者更根本、更可靠。

### 2.4 修正 warpfield 的風險

修正 ear_offset 後，訓練資料的 GT binaural 是用 KEMAR 錄製的（ITD=30.1 samples），
而 GeoWarp 的初始 ITD 也變成 30.1 samples，兩者一致。

**風險**：如果訓練資料的 GT 實際上不是 KEMAR 的 ITD（例如資料集有其他處理），
修正後可能反而變差。建議先在 test_static 上驗證 GeoWarp 的 ITD 是否與 GT 一致。

## 3. FiLM ResBlock 注入 IPD 的架構設計

### 3.1 三個選項的比較

**選項 A：計算 geometric ITD，作為額外 conditioning 注入 FiLM**

```python
# 在 forward() 中計算 geometric ITD
warpfield = self.geo_warper._warpfield(view, T)  # (B, 2, T)
itd = warpfield[:, 0, :] - warpfield[:, 1, :]   # (B, T)，左耳延遲 - 右耳延遲
itd_mean = itd.mean(dim=-1, keepdim=True)        # (B, 1)，時間平均 ITD

# 將 ITD 注入 FiLM conditioning
itd_feat = self.itd_encoder(itd_mean)            # (B, itd_dim)
pos_feat = torch.cat([pos_feat, itd_feat], dim=-1)  # 擴展 conditioning
```

**優點**：
- ITD 是一個**物理量**，直接告訴模型「左右耳應該有多少延遲差」
- 梯度可以從 FiLM 的 gamma/beta 流回 ITD encoder，強化 ITD 的影響
- 不需要修改 loss 函數

**缺點**：
- 需要修改 FiLM conditioning 的維度（pos_dim 從 256 增加）
- ITD 已經隱含在 y_init 的相位中，可能是冗餘資訊

**選項 B：在 loss 中加入 IPD consistency loss（已試過，效果有限）**

v5b 已驗證：即使 w_ipd = w_phase = 1.0，val_ipd 仍在 2.2-2.4。
根本原因是梯度競爭（見第 1.4 節），不是 weight 不夠大。

**選項 C：修改 warpfield 本身（ear_offset 修正）**

見第 2 節分析。這是最根本的修正。

### 3.2 推薦方案：選項 C + 選項 A 組合

單獨使用選項 C（修正 warpfield）可以解決 GeoWarp ITD 低估的問題，
但 FiLM ResStack 仍然可能用 phase residual 抹掉 ITD。

因此建議**同時使用選項 C 和選項 A**：
1. 修正 ear_offset → GeoWarp 輸出正確的 ITD
2. 將 geometric ITD 顯式注入 FiLM conditioning → 告訴模型「不要改變 ITD」

### 3.3 FiLM 注入 IPD 的具體設計

根據 Perez et al. (2018) 的 FiLM 原始設計，conditioning 信號應該是**語義豐富的特徵**，
而不是原始數值。

建議設計：

```python
class ITDConditioningEncoder(nn.Module):
    """
    將 geometric ITD 編碼為 FiLM conditioning 的一部分。
    ITD 是一個時間序列 (B, T)，需要壓縮為固定維度向量。
    """
    def __init__(self, itd_dim=32):
        super().__init__()
        # 用 1D conv 提取 ITD 的統計特徵
        self.encoder = nn.Sequential(
            nn.Conv1d(1, 16, kernel_size=7, padding=3),
            nn.ReLU(),
            nn.AdaptiveAvgPool1d(8),   # 壓縮到 8 個時間步
            nn.Flatten(),
            nn.Linear(16 * 8, itd_dim),
        )

    def forward(self, itd):
        # itd: (B, T)
        return self.encoder(itd.unsqueeze(1))  # (B, itd_dim)
```

在 `GeoWarpFiLMNet.forward()` 中：
```python
# 計算 geometric ITD（samples）
warpfield = self.geo_warper._warpfield(view, T)  # (B, 2, T)
geo_itd = warpfield[:, 0, :] - warpfield[:, 1, :]  # (B, T)

# 編碼 ITD
itd_feat = self.itd_encoder(geo_itd)  # (B, 32)

# 合併到 pos_feat
pos_feat = torch.cat([pos_feat_from_fourier, itd_feat], dim=-1)  # (B, 288)
```

**設計理由**：
- 使用 1D Conv 而非直接 Linear，因為 ITD 是時間序列，有局部結構
- AdaptiveAvgPool 壓縮時間維度，保留 ITD 的統計特性
- 維度 32 足夠表達 ITD 的變化範圍（±30 samples），不會過度增加參數量

## 4. IPD Loss 設計問題

### 4.1 現有 IPD loss 的根本缺陷

v5b 的 `ipd_loss_fn` 計算：
```python
pred_ipd = torch.angle(Y_L * Y_R.conj())   # (B, F, T)
gt_ipd   = torch.angle(Y_L_gt * Y_R_gt.conj())
diff = atan2(sin(pred_ipd - gt_ipd), cos(pred_ipd - gt_ipd))
loss = (diff[mask] ** 2).mean()
```

**問題 1：全頻段平均**

IPD 在不同頻段的物理意義不同：
- 低頻（< 1500 Hz）：IPD 是 ITD 的線性函數，直接反映空間位置
- 高頻（> 1500 Hz）：IPD 有 ambiguity（相位折疊），不能直接用來估計 ITD

現有 loss 對所有頻段平均，高頻的 ambiguous IPD 會稀釋低頻的有效梯度。
根據 Stern & Trahiotis (1995) 的研究，ITD 的感知主要依賴 < 1500 Hz 的低頻 IPD。

**問題 2：MSE 而非 circular distance**

`diff ** 2` 是 MSE，但相位差是循環量（circular variable）。
當 pred_ipd 和 gt_ipd 分別在 +π 和 -π 附近時，
MSE 會給出很大的誤差，但實際的 circular distance 很小。

雖然 `atan2(sin, cos)` 已經做了 wrapping，但 `** 2` 仍然會放大大誤差，
導致梯度被少數 outlier 主導。

**問題 3：能量 mask 的閾值設計**

```python
mask = energy > 0.1 * energy.mean()
```

這個 mask 在靜音段（energy ≈ 0）時幾乎所有頻率都被 mask 掉，
導致 IPD loss ≈ 0，沒有梯度。但靜音段的 IPD 其實是最容易學習的（GT IPD 也接近 0）。

### 4.2 改進方案：低頻加權 IPD loss

參考 BinauralGrad (Chen et al., 2022) 的做法，他們在 diffusion model 的 conditioning 中
使用了頻率加權的 IPD 特徵，低頻 IPD 的權重更高。

參考 DPATFNet (Luo et al., 2022) 的 dual-path attention，
他們在時頻域分別處理 ITD（低頻相位）和 ILD（高頻幅度）。

建議的改進 IPD loss：

```python
def ipd_loss_fn_v6(Y_L, Y_R, Y_L_gt, Y_R_gt, low_freq_bins=64):
    """
    低頻加權 IPD loss。
    - 只用低頻 bins（< 1500 Hz，約前 64 bins @ n_fft=1024, sr=48000）
    - 使用 cosine distance 而非 MSE，對 outlier 更穩健
    - 能量加權：高能量頻率的 IPD 誤差更重要
    """
    # 只取低頻
    Y_L_low    = Y_L[:, :low_freq_bins, :]      # (B, K, T)
    Y_R_low    = Y_R[:, :low_freq_bins, :]
    Y_L_gt_low = Y_L_gt[:, :low_freq_bins, :]
    Y_R_gt_low = Y_R_gt[:, :low_freq_bins, :]

    # Cross-spectrum（複數形式，保留梯度）
    pred_cross = Y_L_low * Y_R_low.conj()       # (B, K, T)
    gt_cross   = Y_L_gt_low * Y_R_gt_low.conj()

    # 能量加權
    energy = gt_cross.abs().clamp(min=1e-8)
    weight = energy / energy.mean()

    # Cosine distance（等價於 1 - cos(phase_diff)，對 outlier 穩健）
    # cos(phase_diff) = Re(pred_cross * conj(gt_cross)) / (|pred_cross| * |gt_cross|)
    numerator   = (pred_cross * gt_cross.conj()).real
    denominator = pred_cross.abs() * gt_cross.abs() + 1e-8
    cos_sim = numerator / denominator           # (B, K, T)，範圍 [-1, 1]

    # loss = 1 - cos_sim（0 = 完美，2 = 最差）
    loss = (1.0 - cos_sim) * weight
    return loss.mean()
```

**改進理由**：
1. 只用低頻 bins：消除高頻 ambiguity 的干擾
2. Cosine distance：對相位 outlier 更穩健（bounded loss）
3. 能量加權：高能量頻率的 IPD 更可靠，給予更高權重

### 4.3 與 BinauralGrad 和 DPATFNet 的比較

**BinauralGrad (Chen et al., 2022)**：
使用 diffusion model，conditioning 包含 mono 信號和空間特徵（包含 IPD）。
他們的 IPD 特徵是從 GT binaural 計算的，作為 conditioning 輸入，
而不是作為 loss 的一部分。這避免了梯度競爭問題。

**DPATFNet (Luo et al., 2022)**：
使用 dual-path attention，分別在時域和頻域處理空間特徵。
他們的 position attention 模組直接學習 ITD 和 ILD 的映射，
不依賴 phase residual 架構，因此沒有 ITD 被抹掉的問題。

**本專案的限制**：
phase residual 架構（GeometricWarper → STFT → phase residual head）
本質上允許模型「選擇」是否保留 ITD。
要根本解決這個問題，需要**架構層面的約束**，而不只是 loss 層面的懲罰。

## 5. Warpfield 修正 vs Loss 修正：梯度流動角度

### 5.1 梯度流動路徑比較

**修正 warpfield（改 ear_offset）**：
```
ear_offset（常數）→ displacement → warpfield → y_init → STFT → FiLM → output
                                                ↑
                                         前向傳播修正，不需要梯度
```

修正 ear_offset 是**前向傳播的修正**：
- 直接改變 y_init 的 ITD
- FiLM ResStack 看到的輸入特徵就已經有正確的 ITD
- 模型只需要「保留」ITD，不需要「學習」ITD

**修正 loss（加強 IPD loss）**：
```
output → IPD loss → ∂L/∂phase_residual → phase_head → FiLM → ...
                         ↑
                  反向傳播修正，受梯度競爭影響
```

修正 loss 是**反向傳播的修正**：
- 需要梯度從 output 反傳到 phase residual head
- Phase loss 的梯度（∂L_phase/∂θ）和 IPD loss 的梯度（∂L_ipd/∂θ）方向不同
- 根據 Yu et al. (2020) 的 PCGrad 分析，當兩個梯度夾角 > 90° 時，
  相加後的合梯度比單個梯度都小，導致優化效率極低

### 5.2 實驗證據

v5b 的訓練曲線顯示：
- Stage 2 Ep 1：val_ipd = 2.357
- Stage 2 Ep 17：val_ipd = 2.265
- 17 個 epoch 只改善了 0.092（4%）

這個緩慢的改善速度符合梯度競爭的預測：
IPD loss 的梯度被 Phase loss 的梯度部分抵消，有效學習率極低。

### 5.3 結論

修正 warpfield 比加強 IPD loss **更根本**，原因：
1. 不依賴梯度傳播，不受梯度競爭影響
2. 直接修正物理模型的系統性誤差
3. 為後續的 FiLM 學習提供正確的初始條件

但修正 warpfield 後，仍需要防止 phase residual 抹掉 ITD。
這需要架構層面的約束（選項 A：顯式注入 ITD conditioning）。

## 6. v6 架構建議

### 6.1 修改清單

#### 修改 1：修正 ear_offset（最高優先級）

**檔案**：`src/models.py`，`GeometricWarper._3d_displacements()`

```python
# 修改前
left_ear_offset  = th.Tensor([0, -0.08, -0.22])
right_ear_offset = th.Tensor([0,  0.08, -0.22])

# 修改後（KEMAR 標準耳間距 0.215m，每側 0.1075m）
left_ear_offset  = th.Tensor([0, -0.1075, -0.22])
right_ear_offset = th.Tensor([0,  0.1075, -0.22])
```

**預期效果**：±90° 的 GeoWarp ITD 從 22.4 samples 增加到 30.1 samples，
與 GCC-PHAT 的 ear_distance=0.215m 完全一致。

**驗證方法**：修改後先跑 `experiments/check_90deg_itd.py`，
確認 GeoWarp 在 ±90° 的 ITD = ±30 samples。

#### 修改 2：FiLM 注入 geometric ITD conditioning

**檔案**：`src/models_geowarp_film_v4.py`（或新建 `models_geowarp_film_v6.py`）

在 `GeoWarpFiLMNet` 中加入 `ITDConditioningEncoder`：

```python
class ITDConditioningEncoder(nn.Module):
    def __init__(self, itd_dim=32):
        super().__init__()
        self.encoder = nn.Sequential(
            nn.Conv1d(1, 16, kernel_size=7, padding=3),
            nn.ReLU(),
            nn.AdaptiveAvgPool1d(8),
            nn.Flatten(),
            nn.Linear(128, itd_dim),
            nn.Tanh(),  # 正規化輸出範圍
        )

    def forward(self, itd):
        # itd: (B, T)，單位 samples，範圍約 [-30, +30]
        itd_norm = itd / 30.0  # 正規化到 [-1, 1]
        return self.encoder(itd_norm.unsqueeze(1))  # (B, itd_dim)
```

在 `GeoWarpFiLMNet.__init__()` 中：
```python
self.itd_encoder = ITDConditioningEncoder(itd_dim=32)
# pos_dim 從 256 增加到 288（256 + 32）
self.pos_encoder = FourierPositionEncoder(7, fourier_L, pos_dim=256)
# FiLM 的 conditioning 維度改為 288
self.res_blocks = nn.ModuleList([
    FiLMResBlock(channels, pos_dim=288, num_bands, dilations[i])
    for i in range(num_blocks)
])
```

在 `GeoWarpFiLMNet.forward()` 中：
```python
# 計算 geometric ITD
warpfield = self.geo_warper._warpfield(view, T)  # (B, 2, T)
geo_itd = warpfield[:, 0, :] - warpfield[:, 1, :]  # (B, T)

# 編碼 ITD
itd_feat = self.itd_encoder(geo_itd)  # (B, 32)

# 合併 conditioning
pos_feat = self.pos_encoder(view)     # (B, 256)
pos_feat = torch.cat([pos_feat, itd_feat], dim=-1)  # (B, 288)
```

**設計理由**（參考 Perez et al., 2018）：
FiLM 的 conditioning 信號應該包含所有與輸出相關的語義信息。
geometric ITD 是一個強烈的空間線索，顯式注入後，
FiLM 的 gamma/beta 可以學習「根據 ITD 調整頻率特徵」，
而不是讓 phase residual 自由選擇是否保留 ITD。

#### 修改 3：低頻加權 IPD loss

**檔案**：`training/train_geowarp_film_v6.py`

用 `ipd_loss_fn_v6`（見第 4.2 節）替換現有的 `ipd_loss_fn`。

關鍵參數：
- `low_freq_bins=64`：對應 n_fft=1024 時的 0~1500 Hz（64/513 × 24000 ≈ 3000 Hz，
  實際應設為 `int(1500 * n_fft / sample_rate) = int(1500 * 1024 / 48000) = 32`）
- 使用 cosine distance 而非 MSE

**注意**：n_fft=1024, sr=48000 時，每個 bin = 48000/1024 ≈ 46.9 Hz，
1500 Hz 對應 bin 32。建議 `low_freq_bins=32`。

#### 修改 4：Phase residual 的 detach 策略調整

現有程式碼：
```python
phase_L_geo = torch.angle(Y_L_init).detach()  # ← detach 阻斷梯度
phase_R_geo = torch.angle(Y_R_init).detach()
phase_L = phase_L_geo + phase_L_res
phase_R = phase_R_geo + phase_R_res
```

**問題**：`detach()` 阻斷了從 phase loss 到 GeoWarp 的梯度，
這是正確的（GeoWarp 是固定的物理模型），但也意味著
phase residual head 必須獨立學習所有的相位修正。

**建議**：保持 `detach()`，但在 phase residual head 的輸出加入一個
**ITD-preserving constraint**：

```python
# 計算 phase residual 的 IPD 分量
phase_res_ipd = phase_L_res - phase_R_res  # (B, F, T)

# 懲罰 phase residual 改變 IPD（只懲罰低頻）
K = 32  # 低頻 bins
ipd_preservation_loss = (phase_res_ipd[:, :K, :] ** 2).mean()
```

這個 loss 直接懲罰 phase residual 改變低頻 IPD，
梯度路徑短（直接到 phase_head），不受 Phase loss 干擾。

### 6.2 訓練策略調整

| Stage | Loss | 權重 | 目的 |
|-------|------|------|------|
| Stage 1 (20 ep) | mag_anchor + phase | 0.1 + 0.1 | 建立基礎幅度和相位 |
| Stage 2 (80 ep) | L2 + Phase + IPD_v6 + IPD_preserve | w_l2 + 1.0 + 0.5 + 0.3 | 全局優化，保護 ITD |

**Stage 2 的 IPD_preserve loss**：
```python
# 在 run_epoch 中加入
ipd_preserve = (phase_res[:, 0, :32, :] - phase_res[:, 1, :32, :]).pow(2).mean()
loss = w_l2*l2 + 1.0*phase + 0.5*ipd_v6 + 0.3*ipd_preserve
```

### 6.3 預期效果

| 指標 | Meta 3-block | v4 | v5b | v6 預期 |
|------|-------------|-----|-----|---------|
| Phase (testset) | 0.804 | 0.831 | 0.853 | ~0.84（略退步） |
| Angular error (mean) | **4.0°** | 19.1° | ~19° | **< 8°** |
| subject1 (-90°) | 0.0° | 66.5° | ~66° | < 20° |
| subject7 (+90°) | 0.0° | 0.0° | ~0° | < 5° |

**預期改善的原因**：
1. ear_offset 修正：GeoWarp ITD 從 22.4 → 30.1 samples，直接改善 ±90° 的角度估計
2. ITD conditioning：FiLM 學習根據 ITD 調整特徵，不再自由抹掉 ITD
3. 低頻 IPD loss：梯度集中在有效頻段，減少高頻 ambiguity 的干擾
4. IPD preservation loss：直接約束 phase residual 不改變低頻 IPD

**為什麼 Phase 可能略退步**：
IPD preservation loss 限制了 phase residual 的自由度，
模型無法再用「兩耳同步移相」的方式最小化 Phase loss，
因此 Phase 可能從 0.853 退步到 ~0.84。
這是可接受的 trade-off（Phase 退步 0.01，Angular error 改善 10°+）。

## 7. 文獻引用

- **Chandler & Grantham (1992)**：MAA ~1-3°，MAMA ~8.8° at 10°/s。
  目前 v5b 的 angular error ~19° 遠超 MAMA 標準，需要顯著改善。

- **Richard et al. (2022)**（Meta Binaural）：GeometricWarper + WaveNet 架構，
  3-block 版本達到 angular error 4.0°。他們的 WaveNet 在波形域操作，
  ITD 由 warpfield 硬編碼，phase residual 問題不存在。

- **Chen et al. (2022)**（BinauralGrad）：Two-stage diffusion model，
  Stage 1 生成 mono-to-binaural 的粗略版本，Stage 2 用 diffusion 精修。
  他們的 IPD conditioning 是從 GT 計算的，作為 diffusion 的 guidance，
  不是作為 loss，避免了梯度競爭。

- **Perez et al. (2018)**（FiLM）：Feature-wise Linear Modulation，
  conditioning 信號通過 gamma/beta 調製特徵圖。
  關鍵洞察：conditioning 應包含所有與輸出相關的語義信息。
  本報告建議將 geometric ITD 加入 conditioning，符合此原則。

- **Sener & Koltun (2018)**（Multi-Task Learning as Multi-Objective Optimization）：
  多目標優化的 Nash 均衡分析，解釋了為什麼 Phase loss 和 IPD loss 會競爭。

- **Yu et al. (2020)**（PCGrad）：梯度競爭分析，當兩個 loss 的梯度夾角 > 90° 時，
  相加後的合梯度比單個梯度都小。建議使用 PCGrad 或 GradNorm 處理梯度競爭。

- **Keskar et al. (2017)**（On Large-Batch Training for Deep Learning）：
  SGD 傾向於收斂到 flat minimum（寬廣低谷），解釋了為什麼模型選擇 ITD=0 的解。

- **Stern & Trahiotis (1995)**：ITD 的感知主要依賴 < 1500 Hz 的低頻 IPD，
  支持低頻加權 IPD loss 的設計。
