# DPATFNet 路線架構分析與設計

**日期**：2026-04-28  
**目標**：解決 ImprovedResidualPhaseNet 的 Phase 學習失敗問題，向 DPATFNet（ICASSP 2025 SOTA）靠攏

---

## 1. 現況診斷

### 公平比較結果（Meta 原版 loss）

| 模型 | L2 (×10³) | Amplitude | Phase |
|------|-----------|-----------|-------|
| Meta small (1 block) | 0.197 | 0.043 | 0.862 |
| Meta large (3 block) | 0.144 | 0.036 | 0.804 |
| DPATFNet (ICASSP 2025) | 0.148 | 0.037 | **0.717** |
| ImprovedResidualPhaseNet (本專案) | 0.628 | 0.045 | 1.553 |

Phase 1.553 ≈ π/2，代表相位接近隨機，模型完全沒有學到空間定位。

## 2. ImprovedResidualPhaseNet 問題根因

### 2.1 架構

```
mono → STFT → mono_mag, mono_phase
ViewEncoder (Linear mean-pool) → view_feat
MagnitudeNet (Conv1d) → mag_L, mag_R
Woodworth(atan2(y,x)) → physical_itd   ← 問題①
ResidualITDNet (Conv1d×2) → residual   ← 問題②
phase = mono_phase ± 2π×freq×(itd+residual)
iSTFT(mag × exp(i×phase)) → y_L, y_R
```

### 2.2 三個根本問題

| # | 問題 | 影響 |
|---|------|------|
| ① | Woodworth 用 `atan2(y,x)`，無 quaternion rotation | ITD baseline 不準，residual 要補的洞太大 |
| ② | ResidualITDNet 只有 2 層 Conv1d，無 attention | 無法建模 position-phase 非線性關係 |
| ③ | Phase 只靠 waveform L2 → iSTFT 間接梯度 | 梯度路徑週期性，難收斂 |

---

## 3. DPATFNet 架構（ICASSP 2025）

```
mono
 │
 ├─ GeometricWarper（Meta 完整幾何）
 │     quaternion rotation + mouth offset + ear offset
 │     → y_init (B,2,L)   ← ITD 已對齊
 │
 ├─ STFT → Y_init (B, 2F, T)
 │
 ├─ DPAB（Dual Position Attention Block）
 │     PAL1: Cross-Attention(position q, orientation k/v) → cond1
 │     PAL2: Self-Attention(cond1) → cond2（捕捉位置變化時刻）
 │     cond = Concat(Conv(cond2), cond0)
 │
 ├─ MPF（Moving Position Fusion）：cond 注入頻域特徵
 │
 ├─ Upsampling（ConvTranspose + ResStack，每層注入 cond0）
 │
 └─ Conv1 + Tanh → y_L, y_R

Loss = λ1×L2 + λ2×PhaseLoss
```

---

## 4. 差異對照

| 面向 | ImprovedResidualPhaseNet | DPATFNet |
|------|--------------------------|----------|
| ITD 初始化 | Woodworth（無 quaternion） | GeometricWarper（完整幾何） |
| Phase 修正 | Conv1d residual | Cross-Attention + Self-Attention |
| 位置條件注入 | ViewEncoder mean-pool（丟失時序） | 每層 ResStack 注入，保留時序 |
| Doppler effect | 無 | PAL2 Self-Attention 捕捉位置變化 |
| Phase 直接監督 | 無 | PhaseLoss |
| IPD 監督 | 無 | 無（PhaseLoss 隱含） |

## 5. 新架構設計

### 5.1 核心思路

- **Time domain**：用 Meta `GeometricWarper` 做完整幾何 ITD（取代 Woodworth）
- **Frequency domain**：用 DPAB Attention 修正 IPD（取代 Conv1d residual）
- **Loss**：加 IPD loss 直接監督相位差（SL-NN 類別）

### 5.2 架構草圖

```
mono (B,1,L)  +  view (B,7,K)
        │
        ├─ GeometricWarper → y_init (B,2,L)
        │
        ├─ STFT → Y_init (B, 2×F, T)
        │
        ├─ DPAB(view) → cond (B, 10, K)
        │     PAL1: Cross-Attn(pos, orient)
        │     PAL2: Self-Attn（位置變化）
        │
        ├─ MPF：cond 插值後注入 Y_init
        │
        ├─ ResStack × N（每層注入 cond0）
        │
        └─ Conv1 + Tanh → y_L, y_R (B,2,L)

Loss = λ1×L2 + λ2×PhaseLoss + λ3×IPD_loss
```

### 5.3 分階段訓練

| Stage | 凍結 | 訓練 | Loss |
|-------|------|------|------|
| 1 | — | DPAB + ResStack | IPD_loss（直接監督相位差） |
| 2 | — | 全部 | λ1×L2 + λ2×PhaseLoss + λ3×IPD_loss |

Stage 1 讓 phase 先收斂到合理範圍，避免 waveform L2 的不穩定梯度。

### 5.4 目標指標

| 指標 | 現況 | 目標 |
|------|------|------|
| Phase | 1.553 | < 0.804（超越 Meta large） |
| L2 (×10³) | 0.628 | < 0.197（超越 Meta small） |
| Amplitude | 0.045 | < 0.043 |
| ITD 誤差 | 未測 | < 100 µs（人類 MAA JND） |

## 6. PGCN 提案評估（2026-04-28）

**來源**：`docs/architecture/PGCN_README.md`

### 優點
- Fourier Position Encoder（L=10）：解決 120Hz 離散化精度問題，有理論依據
- Complex-valued 全程：避免 phase wrapping
- 64-band FiLM：比 DPATFNet 的 1-query Cross-Attention 更精細的頻率控制

### 問題
| # | 問題 | 說明 |
|---|------|------|
| ① | Physics Head 還是 Woodworth | `atan2(pos_feat[:,1], pos_feat[:,0])` 對 Fourier 編碼後的向量無意義 |
| ② | 沒有 GeometricWarper | 純頻域從 mono 學 ITD，難度比 DPATFNet 更高 |
| ③ | ComplexConv2d 等需自製 | 訓練穩定性未驗證，每個都是新風險點 |
| ④ | Minimum-Phase 實作有 bug | Hilbert transform 實作不正確 |
| ⑤ | 過度工程化 | 24M 參數、6 項 loss，baseline 未驗通就上複雜架構 |

### 結論
PGCN 的好想法（Fourier Encoder + 64-band FiLM）值得保留，但整體架構風險過高。

## 8. 決策：進入混合方案

根據 HybridTFNet 失敗結果，不再等待，直接進入新架構。

### 新架構：GeometricWarper + Fourier Encoder + 64-band FiLM

取 DPATFNet 路線的正確 ITD 起點，加入 PGCN 的高精度位置編碼和精細頻率調制：

```
mono + view
  │
  ├─ GeometricWarper（Meta 完整幾何，含 quaternion）→ y_init (B,2,L)
  │
  ├─ STFT(y_init) → Y_init (B, 2F, T)   ← 左右耳 concat，ITD 已對齊
  │
  ├─ Fourier Position Encoder（L=8）→ pos_feat (B,256)
  │     sin/cos of [2^0...2^7]×π×position
  │
  ├─ Conv1d Encoder → feat (B, C, T)
  │
  ├─ ResStack × N，每層注入 64-band FiLM(pos_feat)
  │     FiLM: gamma/beta per frequency band
  │
  └─ Conv1 + Tanh → y_L, y_R

Loss = λ1×L2 + λ2×PhaseLoss + λ3×IPD_loss
```

### 與各方案比較

| | HybridTFNet | PGCN | 新方案 |

### 10.1 v1 架構與結果（2026-04-28）

**訓練配置**：

- Stage 2 (70 epochs): `loss = w_l2×l2 + w_phase×phase + w_ipd×ipd`

**結果**：
| 指標 | v1 結果 | Meta small | Meta large |
|------|---------|-----------|-----------|
| L2 (×10³) | **0.166** | 0.197 | 0.144 |
| Amplitude | 0.046 | 0.043 | 0.036 |
| Phase | **0.863** | 0.862 | 0.804 |

✅ **成就**：L2 超越 Meta small 16%，Phase 與 Meta small 持平

### 10.2 v2 失敗分析（2026-04-29）

**調整**：Stage 1: 30→50 epochs, Stage 2: 70→100 epochs, Patience: 15→20

**結果**：
- Stage 1 Best IPD: 2.1438 (v1: 2.1539) ✅ 更好
- Stage 2 Best Phase: 0.891 (v1: 0.872) ❌ 更差
- Final: L2=0.160e-3, Phase=0.886

**根本原因**：**IPD 過擬合導致 Phase 惡化**

#### 問題診斷

1. **物理先驗的雙面刃**：
   - Stage 1 只優化 IPD（相對相位差），忽略絕對相位
   - 模型學到「左右耳同時偏移 +30°」也能讓 IPD 正確
   - 但 Phase Loss 要求絕對相位準確，產生衝突

2. **IPD vs Phase 的自由度差異**：
   ```
   IPD = angle(Y_L × Y_R*)  → 只關心相位差
   Phase = angle(Y_L), angle(Y_R) → 關心絕對相位
   
   如果 Y_L 和 Y_R 都偏移 θ，IPD 不變，但 Phase 錯誤
   ```

3. **Phase residual 約束過緊**：
   - 限制在 ±π/2，如果 geometric phase 誤差 > π/2，無法修正
   - 模型被迫在有限空間內過擬合 IPD

4. **缺乏時序感知**：
   - `FourierPositionEncoder` 只用 `view.mean(dim=2)`，丟失運動信息
   - 無法捕捉 Doppler effect（DPATFNet 用 Self-Attention 解決）

### 10.3 v3 架構改進（2026-04-29）

**核心改進**：Stage 1 加入弱 Phase 監督

```python
# Stage 1 (v1)
loss = ipd + 0.1 * mag_anchor

# Stage 1 (v3)
loss = ipd + 0.1 * mag_anchor + 0.1 * phase  # 新增
```

**訓練配置**：
- Stage 1 (30 epochs): IPD + mag_anchor + **0.1×phase**
- Stage 2 (80 epochs): L2 + Phase + IPD
- Learning rate: 1e-4, Patience: 20

**預期效果**：
- 防止「IPD 對但絕對相位錯」的陷阱
- Stage 1 同時優化相對相位差和絕對相位
- Stage 2 起點更好，收斂更快

**目標**：Phase < 0.80（超越 Meta large）

### 10.4 v4 規劃：架構升級（待實施）

基於深度分析，v4 將解決根本性架構問題：

#### 改進 1：時序感知 Position Encoder
```python
class FourierPositionEncoder(nn.Module):
    def __init__(self, ...):
        # 新增：1D Conv 捕捉運動模式
        self.temporal_conv = nn.Conv1d(7, 7, kernel_size=5, padding=2, groups=7)
    
    def forward(self, view):
        # view: (B, 7, K)
        view_temporal = self.temporal_conv(view)  # 捕捉速度/加速度
        view_mean = view.mean(dim=2)
        view_temporal_mean = view_temporal.mean(dim=2)
        
        # Fourier + raw + temporal
        fourier_feat = self.fourier_encode(view_mean)
        combined = torch.cat([fourier_feat, view_temporal_mean], dim=1)
        return self.mlp(combined)
```

**好處**：
- 捕捉位置變化軌跡（速度、加速度）
- 模擬 Doppler effect
- 類似 DPATFNet 的 Self-Attention，但更輕量

#### 改進 2：放寬 Phase Residual 約束
```python
# v1-v3: ±π/2
phase_res = self.phase_head(x) * (torch.pi / 2)

# v4: ±π
phase_res = self.phase_head(x) * torch.pi
```

**理由**：
- 如果 geometric phase 誤差 > π/2，模型需要更大修正空間
- 實驗性調整，需監控是否破壞 geometric prior

#### 改進 3：增加 FiLM Blocks
```python
# v1-v3: 6 blocks
self.res_blocks = nn.ModuleList([
    FiLMResBlock(...) for _ in range(6)
])

# v4: 8 blocks
self.res_blocks = nn.ModuleList([
    FiLMResBlock(...) for _ in range(8)
])
```

**理由**：
- 音訊合成需要更深的網絡捕捉細節
- DPATFNet 使用多層 Upsampling + ResStack

#### 改進 4：減少頻段數（可選）
```python
# v1-v3: 64 bands
FiLMLayer(channels, pos_dim, num_bands=64)

# v4: 32 bands
FiLMLayer(channels, pos_dim, num_bands=32)
```

**理由**：
- 64 頻段可能過細，每段訓練樣本稀疏
- 32 頻段在精度和穩定性間平衡

### 10.5 與 DPATFNet 的差距分析

| 面向 | GeoWarpFiLMNet (v3) | DPATFNet | 差距 |
|------|---------------------|----------|------|
| **ITD 初始化** | GeometricWarper (完整幾何) | GeometricWarper | ✅ 相同 |
| **Position 編碼** | Fourier (L=8) | Learned embedding | ≈ 相當 |
| **時序建模** | ❌ 無 | Self-Attention (PAL2) | ⚠️ 關鍵差距 |
| **條件注入** | 64-band FiLM (每層) | Cross-Attention + MPF | ≈ 相當 |
| **網絡深度** | 6 blocks | 多層 Upsampling | ⚠️ 可能不足 |
| **Phase 監督** | Stage 1: IPD+Phase, Stage 2: L2+Phase+IPD | L2 + Phase | ✅ 更強 |
| **結果** | Phase=0.863 | Phase=0.717 | 差距 20% |

**結論**：
- ✅ **物理先驗**：GeometricWarper 提供正確 ITD baseline
- ✅ **條件注入**：64-band FiLM 理論上比 Cross-Attention 更精細
- ❌ **時序建模**：缺乏運動感知是最大短板
- ⚠️ **網絡容量**：6 blocks 可能不足以捕捉細節

**v4 目標**：加入時序感知後，Phase 應能突破 0.80，接近 DPATFNet 的 0.717

  │     每層：Conv2d(dilation) + BN + ReLU + 64-band FiLM(pos_feat)

**關鍵設計**：
- ✅ Phase residual 只修正 ±π/2，避免破壞 geometric prior
- ✅ `.detach()` 防止梯度回傳到 GeometricWarper（避免 sqrt(0) 爆炸）
- ✅ Magnitude clamp 1e-6 防止零值
- ✅ 返回 6 個值：`(y_L, y_R, Y_L, Y_R, Y_L_init, Y_R_init)`

### 10.2 訓練策略

| Stage | Epochs | LR | Loss | Metric |
|-------|--------|----|----|--------|
| 1 | 30 | 1e-4 | `ipd + 0.1×mag_anchor` | val_ipd |
| 2 | 70 | 1e-4 | `w_l2×l2 + w_phase×phase + w_ipd×ipd` | val_phase |

**Loss 定義**：
- **IPD loss**: `mean((angle(Y_L×Y_R*) - angle(Y_L_gt×Y_R_gt*))²)`，能量 mask > 0.1×mean
- **Magnitude anchor**: `L1(|Y_L| - |Y_L_init|)`，防止 Stage 1 偏離 geometric prior
- **L2 loss**: `mean((y_pred - y_gt)²)` (Meta 原版)
- **Phase loss**: `mean(|angle(Y_pred) - angle(Y_gt)|)` (Meta 原版)

**Weight calibration**（Stage 2 開始前）：
```python
w_l2    = phase_raw / (l2_raw + 1e-8)     # ~1000
w_phase = 1.0                              # 基準
w_ipd   = phase_raw / (ipd_raw + 1e-8)    # ~0.5
```

### 10.3 數值穩定性修正（關鍵）

訓練初期遇到 NaN 爆炸，經 4-agent code review 診斷出根因：

| 位置 | 問題 | 修正 |
|------|------|------|
| `src/warping.py:100` | `distance = (x²+y²+z²)**0.5` → sqrt(0) 梯度 ∞ | `sqrt(x²+y²+z² + 1e-8)` |
| `src/losses.py:55` | `mag = (real²+imag²)**0.5` → sqrt(0) 梯度 ∞ | `sqrt(x.clamp(min=1e-12))` |
| `models_geowarp_film.py:243` | `angle(Y_init)` 梯度回傳到 warper | `angle(Y_init).detach()` |
| `models_geowarp_film.py:236` | `mag_out` 可能為 0 | `mag_out.clamp(min=1e-6)` |
| `train_geowarp_film.py:48` | IPD loss 可能 NaN | 加 `torch.isnan()` guard |

**根本原因**：`sqrt(0)` 的梯度是 `1/(2*sqrt(0)) = ∞`，在 `clip_grad_norm_` 之前就污染了 Adam optimizer 的 momentum。

**Meta 原版也有這個 bug**，但他們的 WaveNet 不對 geometric phase 求導，所以沒觸發。
