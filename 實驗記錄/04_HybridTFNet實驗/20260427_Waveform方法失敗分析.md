# 為什麼 Waveform 方法仍然無法學好 Phase？

**日期**: 2026-04-27  
**問題**: WaveformSpatializer 雖然改善了 IPD，但 Phase 仍然接近隨機

---

## 實驗結果

```
Epoch 12:
  Waveform L2: 0.000634
  IPD: 0.8773
  Phase_L: 0.9967  ← 接近 1.0 = 隨機
  Phase_R: 0.9968  ← 接近 1.0 = 隨機
```

**對比 DPATFNet**:
```
DPATFNet:
  L2: 0.000719
  Phase: 3.18 (wrapped cosine loss)
```

**換算**:
```
Phase loss = 1 - cos(θ)
0.9967 → cos(θ) ≈ 0.0033 → θ ≈ 89.8° ≈ 1.57 rad

這仍然是很大的相位誤差！
```

---

## 根本原因分析

### 1. 時域 Filter 的限制

**我們的模型**:
```python
filter_L = FilterNet(view)  # B×1×512 (10.7ms @ 48kHz)
filter_R = FilterNet(view)  # B×1×512

binaural = conv1d(mono, filter)
```

**問題**:
- Filter 長度只有 512 samples (10.7ms)
- 這只能捕捉短時相位關係
- 無法建模長時相位演化

**HRTF 的真實特性**:
- 低頻 ITD: 需要長時延遲 (±0.6ms)
- 高頻相位: 複雜的頻率相關變化
- 512 samples 不夠長

### 2. 卷積的線性限制

**時域卷積 = 線性時不變系統**:
```
y(t) = ∫ h(τ) x(t-τ) dτ

這只能建模:
- 線性相位延遲
- 固定的頻率響應
```

**但真實 HRTF**:
- 可能有非線性效應
- 頻率相關的相位變化
- 方向相關的複雜模式

### 3. Loss 權重問題

**當前配置**:
```python
loss = 1.0 × Waveform_L2
     + 1.0 × Magnitude
     + 1.0 × IPD
```

**問題**:
- Waveform L2 主導了梯度
- Phase 的貢獻被稀釋
- 模型優先學習 magnitude

**數值分析**:
```
Waveform: 0.000634  (量級 10^-4)
Magnitude: 0.074    (量級 10^-2)
IPD: 0.877          (量級 10^0)

實際梯度貢獻:
Waveform >> Magnitude >> IPD
```

### 4. 資訊理論限制仍然存在

**核心問題沒有改變**:
```
I(Phase; Mono, View) ≈ 0

即使在時域學習，資訊仍然不足
```

**為什麼 IPD 有改善但 Phase 沒有？**
```
IPD = Phase_L - Phase_R

IPD 的約束:
- 只需要相對關係正確
- 可以有全局偏移
- 自由度更少

Phase_L, Phase_R 的約束:
- 需要絕對值正確
- 沒有自由度
- 更難學習
```

## 數學證明

### Waveform Loss 不等於 Phase Loss

**時域 MSE**:
```
L_waveform = E[(y_pred - y_gt)²]
           = E[(A_pred cos(ωt + φ_pred) - A_gt cos(ωt + φ_gt))²]
```

**展開**:
```
≈ E[(A_pred - A_gt)²] + E[A² (φ_pred - φ_gt)²]
  ^^^^^^^^^^^^^^^^      ^^^^^^^^^^^^^^^^^^^^^^^^
  Magnitude error       Phase error (weighted by A²)
```

**關鍵**:
- Phase error 被 amplitude 加權
- 低能量頻段的 phase 被忽略
- 但這些頻段對空間感知很重要

### 為什麼 IPD 改善了？

**IPD 的特殊性**:
```
IPD = angle(Y_L / Y_R)

在時域:
y_L(t) = h_L * x(t)
y_R(t) = h_R * x(t)

如果 h_L 和 h_R 有時間差 Δt:
IPD ≈ ω × Δt

這是線性關係，容易學習
```

**但個別 Phase**:
```
Phase_L = angle(Y_L)
        = angle(H_L × X)
        = angle(H_L) + angle(X)

需要同時學習:
1. H_L 的相位
2. 與 X 的相位交互
3. 頻率相關的變化

太複雜了
```

## 為什麼這個方向可能是死路

### 1. 時域方法的本質限制

**優點**:
- 避開 Phase wrapping
- 直接優化聽覺目標

**缺點**:
- 無法精確控制頻域相位
- Filter 長度限制
- 線性卷積的表達能力有限

### 2. 與 DPATFNet 的對比

| 特性 | DPATFNet | WaveformSpatializer |
|------|----------|---------------------|
| L2 | 0.000719 | 0.000634 (略好) |
| Phase | 3.18 | 0.9967 (仍然差) |
| 方法 | STFT | Waveform |
| 結論 | Phase 學不會 | Phase 還是學不會 |

**本質相同**:
- 都無法學習 Phase
- 只是表現形式不同
- 根本問題沒解決

### 3. 資訊理論的鐵律

```
無論用什麼架構:
- STFT-based
- Waveform-based
- Transformer
- Diffusion

只要 I(Phase; Input) ≈ 0
就學不會 Phase

這是數學限制，不是工程問題
```

## 正確的方向

### 承認現實

**Phase 從 mono 無法預測**:
1. 不是架構問題
2. 不是 loss 問題
3. 是資訊理論限制

### 可行的方案

#### 方案 A: Magnitude-only + 物理 ITD
```python
# 學習 ILD (magnitude)
ILD = NeuralNet(mono, view)

# 物理計算 ITD (phase)
ITD = head_radius × sin(azimuth) / sound_speed

# 合併
binaural = apply_ILD_ITD(mono, ILD, ITD)
```

**優點**:
- ILD 可以學習 ✅
- ITD 用物理模型 ✅
- 結合學習和先驗知識

#### 方案 B: 使用真實 HRTF 數據庫
```python
# 不從 mono 預測，而是從數據庫查詢
HRTF = query_database(view)
binaural = convolve(mono, HRTF)
```

**優點**:
- 完全準確的 Phase ✅
- 不需要學習 Phase
- 只需要學習 view → HRTF 的映射

#### 方案 C: 降低期望
```python
# 接受 Phase 不完美
# 專注於 ILD 和粗略的 ITD
```

**優點**:
- 實際可行 ✅
- 仍然有空間感
- 誠實的科學態度

## 建議

### 立即停止當前方向

**原因**:
1. Waveform 方法沒有根本性突破
2. Phase 仍然學不好
3. 繼續訓練不會有質的改變

### 重新定義問題

**不要問**: "如何學習 Phase？"  
**應該問**: "如何在不完美 Phase 下實現空間化？"

### 實用的目標

```
目標 1: ILD 學習 (magnitude)
  → 可行 ✅

目標 2: 粗略的 ITD (低頻 phase)
  → 用物理模型 ✅

目標 3: 完美的 Phase
  → 不可行 ❌
  → 放棄這個目標
```

## 結論

**WaveformSpatializer 的失敗證明了**:

1. **時域學習不是銀彈**
   - 仍然無法解決資訊不足的問題
   - Phase 學習的根本限制沒有改變

2. **IPD 改善是假象**
   - 0.8773 看起來比 3.18 好
   - 但換算成角度仍然很大
   - 實際聽覺效果可能差不多

3. **需要改變策略**
   - 不要試圖學習 Phase
   - 結合物理模型和學習
   - 降低對 Phase 的期望
