# 為什麼「分兩塊」的訓練方式一直失敗？

**日期**: 2026-04-27  
**核心問題**: Time-Frequency 雙分支架構為何無法學習相位？

---

## 失敗的架構模式

### 我們嘗試的「分兩塊」設計

```python
# HybridTFNet 架構
class HybridTFNet:
    def __init__(self):
        self.shared_encoder = ...      # 共享特徵提取
        self.freq_branch = ...          # 頻域分支 → Magnitude
        self.time_branch = ...          # 時域分支 → Phase
    
    def forward(self, mono, view):
        shared_feat = self.shared_encoder(mono, view)
        
        # 分支 1: 頻域處理 magnitude
        magnitude = self.freq_branch(shared_feat)
        
        # 分支 2: 時域處理 phase
        phase = self.time_branch(shared_feat)
        
        # 合併
        Y = magnitude * exp(j * phase)
        return Y
```

### 為什麼這個設計看起來合理？

**理論動機**:
1. **Magnitude 是頻域特性** - 能量分佈、共振峰
2. **Phase 是時域特性** - 時間延遲、群延遲
3. **分開處理** - 讓每個分支專注於自己的任務
4. **類似人類聽覺** - 雙耳系統分別處理 ILD 和 ITD

**文獻支持**:
- 許多語音增強模型使用類似架構
- Magnitude-Phase 分離是經典方法
- 看起來是「最佳實踐」

## 為什麼實際上失敗了？

### 問題 1: Phase 根本不可學

```
核心矛盾:
- 我們假設: Phase 可以從 mono + view 學習
- 實際情況: I(Phase; Mono, View) ≈ 0

不管你用什麼架構，學不到就是學不到
```

**實驗證據**:
```
v1-v6: 各種 Time Branch 設計 → Phase error 3.18
v7: 增加 loss weight → Phase error 3.18  
v8: IPD-only → IPD error 3.07
```

所有變體都失敗，因為**問題不在架構，在任務本身**。

### 問題 2: 分支之間的衝突

```python
# 訓練時發生的事情

# Magnitude branch 學習:
loss_mag = MSE(mag_pred, mag_gt)  # 清晰的梯度 ✅

# Phase branch 嘗試學習:
loss_phase = MSE(phase_pred, phase_gt)  # 混亂的梯度 ❌

# 最終 L2 loss:
loss_l2 = MSE(mag_pred * exp(j*phase_pred), Y_gt)
        = loss_mag + loss_phase
```

**梯度衝突**:
- Magnitude branch 收到有用的梯度 → 正常學習
- Phase branch 收到噪聲梯度 → 隨機遊走
- 兩個分支互相干擾，無法協同優化

### 問題 3: 過度參數化

```
HybridTFNet: 11.5M 參數
- Shared encoder: 3M
- Freq branch: 4M  
- Time branch: 4M
- Output heads: 0.5M

實際需要:
- Magnitude 學習: ~2M 參數就夠了
- Phase 學習: 無限參數也學不會

結果: 8M 參數被浪費在不可能的任務上
```

### 問題 4: 特徵分離的假設錯誤

```
我們的假設:
Magnitude ← Frequency domain features
Phase ← Time domain features

實際情況:
Magnitude ← 可以從任何特徵學習 ✅
Phase ← 無法從任何特徵學習 ❌

分離特徵並不能解決根本問題
```

## 與 DPATFNet 的對比

### DPATFNet 的設計

```python
class DPATFNet:
    def __init__(self):
        self.encoder = ...
        self.decoder = ...
        # 單一路徑，直接預測複數 STFT
    
    def forward(self, mono):
        feat = self.encoder(mono)
        Y_complex = self.decoder(feat)  # 直接輸出複數
        return Y_complex
```

**為什麼 DPATFNet 也失敗？**
- 同樣的問題: Phase 不可學
- 但架構更簡單，參數更少 (515K vs 11.5M)
- 沒有分支衝突，訓練更穩定

### 關鍵差異

| 特性 | HybridTFNet | DPATFNet |
|------|-------------|----------|
| 架構 | 雙分支 (Mag/Phase) | 單路徑 |
| 參數 | 11.5M | 515K |
| 訓練穩定性 | 差 (分支衝突) | 好 |
| Phase 學習 | 失敗 | 失敗 |
| Magnitude 學習 | 成功 | 成功 |

**結論**: 
- 兩者都無法學習 Phase
- DPATFNet 更簡單、更高效
- 複雜架構沒有帶來優勢

## 為什麼「分兩塊」是個陷阱？

### 1. 給人虛假的希望

```
錯覺: "Phase 學不好是因為和 Magnitude 混在一起"
真相: "Phase 學不好是因為根本學不了"

分開處理只是讓問題看起來更清楚，但沒有解決問題
```

### 2. 增加複雜度但沒有收益

```
複雜度成本:
- 2 倍的分支網路
- 2 倍的參數量
- 2 倍的訓練時間
- 分支同步的困難

實際收益:
- Magnitude: 沒有提升 (單分支就能學好)
- Phase: 仍然失敗 (分開也學不會)

ROI = 0 / (2x cost) = 負無窮
```

### 3. 掩蓋了真正的問題

```
如果用單路徑:
→ 很快發現 Phase 學不會
→ 早點放棄，轉向其他方向

如果用雙分支:
→ 不斷調整分支設計
→ 浪費時間在無效的優化上
→ 延遲了發現根本問題
```

## 正確的思考方式

### 不要問: "如何設計架構來學習 Phase？"

這個問題預設了 Phase 可學，但實際上不可學。

### 應該問: "既然 Phase 不可學，如何繞過它？"

**可行的方向**:

#### 1. 放棄端到端學習 Phase
```python
# 方案 A: Magnitude-only
Y_pred = Mag_pred * exp(j * Phase_mono)

# 方案 B: 物理模型 + 學習
ITD = geometric_model(view)  # 幾何計算
ILD = neural_network(mono, view)  # 神經網路
Y_pred = apply_ITD_ILD(Y_mono, ITD, ILD)

# 方案 C: 參數化 HRTF
HRTF_params = neural_network(view)
Y_pred = apply_HRTF(Y_mono, HRTF_params)
```

#### 2. 改變學習目標
```python
# 不學 Phase，學其他東西

# 方案 A: 學習 waveform 直接
y_binaural = WaveNet(y_mono, view)

# 方案 B: 學習 filter
h_HRTF = FilterNet(view)
y_binaural = convolve(y_mono, h_HRTF)

# 方案 C: 學習 residual
y_binaural = y_mono + ResidualNet(y_mono, view)
```

#### 3. 利用先驗知識
```python
# 不從頭學，用已知的物理規律

# 低頻: ITD 主導 (幾何計算)
ITD = head_radius * sin(azimuth) / sound_speed

# 高頻: ILD 主導 (神經網路學習)
ILD = ILD_Net(mono_spectrum, view)

# 合併
Y_pred = frequency_dependent_spatialize(Y_mono, ITD, ILD)
```

## 創新的方向

### ❌ 不要做的「創新」

1. **更複雜的分支設計** - 解決不了根本問題
2. **更深的網路** - Phase 不是容量問題
3. **更多的 loss terms** - 無法改變資訊理論限制
4. **Attention/Transformer** - 架構不是瓶頸

### ✅ 值得嘗試的創新

#### 1. **Waveform-domain 直接學習**
```python
class WaveformSpatializer(nn.Module):
    """直接在時域操作，避開 Phase 問題"""
    
    def forward(self, mono_wave, view):
        # 學習時域 filter
        filter_L = self.filter_net_L(view)
        filter_R = self.filter_net_R(view)
        
        # 時域卷積
        binaural_L = F.conv1d(mono_wave, filter_L)
        binaural_R = F.conv1d(mono_wave, filter_R)
        
        return torch.stack([binaural_L, binaural_R], dim=1)
```

**優點**:
- 避開 Phase wrapping 問題
- 直接優化聽覺目標
- 可以學習非線性效應

**挑戰**:
- 長序列建模
- 計算成本高

#### 2. **Parametric HRTF 學習**
```python
class ParametricHRTF(nn.Module):
    """學習 HRTF 的參數化表示"""
    
    def forward(self, view):
        # 預測 HRTF 參數 (不是直接預測 Phase)
        params = self.param_net(view)
        
        # 用參數生成 HRTF
        HRTF = self.synthesize_HRTF(params)
        
        return HRTF
    
    def synthesize_HRTF(self, params):
        # 用物理模型生成 HRTF
        # 例如: 球形頭模型 + 耳廓共振
        ITD = params['itd']
        ILD = params['ild']
        notch_freq = params['notch_freq']
        ...
```

**優點**:
- 結合物理知識和學習
- 參數空間低維、平滑
- 可解釋性強

**挑戰**:
- 需要設計好的參數化
- 可能無法捕捉所有細節

#### 3. **Frequency-adaptive 策略**
```python
class FrequencyAdaptiveSpatializer(nn.Module):
    """不同頻段用不同方法"""
    
    def forward(self, mono_stft, view):
        # 低頻 (<1kHz): 幾何 ITD
        low_freq = self.apply_geometric_ITD(
            mono_stft[:, :low_cutoff], 
            view
        )
        
        # 中頻 (1-4kHz): 學習 ILD + 簡化 Phase
        mid_freq = self.learn_ILD_simple_phase(
            mono_stft[:, low_cutoff:mid_cutoff],
            view
        )
        
        # 高頻 (>4kHz): 只學習 ILD
        high_freq = self.learn_ILD_only(
            mono_stft[:, mid_cutoff:],
            view
        )
        
        return torch.cat([low_freq, mid_freq, high_freq], dim=1)
```

**優點**:
- 針對不同頻段的特性優化
- 避開最困難的高頻 Phase
- 保留低頻的精確定位

**挑戰**:
- 頻段邊界的平滑過渡
- 多個模組的協調

#### 4. **Implicit Neural Representation**
```python
class INR_HRTF(nn.Module):
    """用 INR 表示 HRTF"""
    
    def forward(self, freq, view):
        # 輸入: 頻率 + 視角
        # 輸出: 該頻率該視角的 HRTF
        
        # 連續表示，沒有離散化問題
        hrtf_value = self.mlp(
            torch.cat([
                self.freq_encoding(freq),
                self.view_encoding(view)
            ])
        )
        
        return hrtf_value
```

**優點**:
- 連續、平滑的表示
- 可以任意解析度查詢
- 自然的插值

**挑戰**:
- 訓練穩定性
- 計算效率

## 總結

### 為什麼「分兩塊」失敗？

1. **Phase 根本不可學** - 架構無法解決資訊理論限制
2. **分支衝突** - Magnitude 和 Phase 的梯度互相干擾
3. **過度複雜** - 增加成本但沒有收益
4. **掩蓋問題** - 延遲了發現根本原因

### 正確的方向

**不要試圖學習 Phase，而是繞過它**:
- Waveform-domain 直接學習
- Parametric HRTF 表示
- Frequency-adaptive 策略
- 結合物理模型和學習

### 創新的關鍵

**不是更複雜的架構，而是更聰明的問題定義**

從 "如何學習 Phase" → "如何不需要學習 Phase"
