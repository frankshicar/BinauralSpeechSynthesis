# BinauralTFNet v8 訓練記錄

## 實驗目標

訓練 BinauralTFNet v8，一個時頻域分離的雙耳音訊合成架構，解決 v7 的梯度衝突問題。

## 架構設計

### 核心思想

將雙耳合成分為兩個獨立的 branch：
- **CommonBranch（時域）**：學習兩耳共同部分，用 WarpNet + WaveNet
- **SpecificBranch（頻域）**：學習左右耳差異，用 STFT + DPAB + TFResStack

### 三階段訓練

| Stage | Epochs | 訓練對象 | Loss |
|-------|--------|---------|------|
| 1 | 0-80 | CommonBranch | L2（純 L2，ITD 只記錄） |
| 2 | 80-160 | SpecificBranch | Phase_L + Phase_R + IPD |
| 3 | 160-180 | 全部參數 | L2×100 + Phase_L + Phase_R + IPD |

### 關鍵設計決策

1. **Warpfield 域平均**：避免信號域平均的 comb filtering
2. **從 warped 出發**：`Y_L = Y_warp_L + delta_L`，保留 Warpnet 的 ITD
3. **Stage 1 純 L2**：ITD loss 量級太大（1.5 vs 0.0001），無法共存

## 失敗原因分析

### 1. **DPAB 實作錯誤（主因）**

**問題**：把整個 view (7 維) 同時投影成位置 (P) 和方向 (O)

```python
# 錯誤實作
v = view.transpose(1, 2)  # B×K×7
pos = self.proj_pos(v)    # Linear(7 → 64)，把全部 7 維當位置
dir_ = self.proj_dir(v)   # Linear(7 → 64)，把全部 7 維當方向
cond1, _ = self.cross_attn(pos, dir_, dir_)  # Q/K/V 語義混亂
```

**正確做法**：分離位置和方向

```python
# view: [x, y, z, quat_w, quat_x, quat_y, quat_z]
pos = view[:, 0:3, :]  # 只取 [x, y, z]
ori = view[:, 3:7, :]  # 只取 [quat_w, quat_x, quat_y, quat_z]

P = self.proj_pos(pos)  # B×K×3 → B×K×64
O = self.proj_ori(ori)  # B×K×4 → B×K×64
cond1, _ = self.cross_attn(P, O, O)  # P 作 Query，O 作 Key/Value
```

**影響**：
- DPAB 無法正確學習位置和方向的交互作用
- Cross-attention 失去語義（Q/K/V 都來自同一個混合特徵）
- 導致 SpecificBranch 無法學到空間資訊

**證據**：
- 預測角度幾乎固定
- Stage 2 改善幅度只有 13%（正常應該 50%+）
- 模型對角度變化有反應（output_diff=0.91），但方向錯誤

### 2. **STFT 解析度不足（次因）**

**當前參數**：
```python
fft_size=512 → F=257
頻率解析度 = 48000/512 = 93.75 Hz
```

**問題**：
- 低頻的 ITD/IPD 資訊需要更高解析度
- 257 個 bins 對於 48kHz 音訊太粗糙
- ITD 主要在 <1500 Hz，需要更精細的頻率劃分

**對比 DPATFNet**：通常用 fft_size=1024 或 2048

### 3. **TFResStack 容量不足（次因）**

**當前參數**：
```python
tf_channels=128
tf_blocks=4
```

**問題**：
- 4 層 ResBlock 太淺，無法學習複雜的頻域變換
- 128 channels 表達能力不足
- 對比 DPATFNet：通常用 8-12 blocks，256-512 channels

### 4. **從 warped 出發的設計可能有問題**

```python
Y_L = Y_warp_L + delta_L  # 從 Warpnet 的輸出出發
```

**假設**：Warpnet 的 ITD 是對的，SpecificBranch 只需要修正細節

**實際**：
- ITD 誤差 772μs（很大）
- 說明 Warpnet 的 warp 本身就不準
- 如果基礎錯了，加 delta 無法完全修正

**可能的改進**：
- 讓 SpecificBranch 有更大的修正能力
- 或者改成從 y_common 出發：`Y_L = Y_common + delta_L`

---

## 結論

v8 的核心架構（時頻域分離）是對的，但 **DPAB 的實作錯誤** 導致 SpecificBranch 無法學到空間資訊。

**關鍵證據**：
- Stage 2 改善幅度只有 13%（正常 50%+）
- 預測角度幾乎固定，沒有方向性
- Phase/ITD/ILD 都是 DPATFNet 的 2-3 倍誤差

**下一步**：v8.2 修正 DPAB 實作，增大模型容量。
