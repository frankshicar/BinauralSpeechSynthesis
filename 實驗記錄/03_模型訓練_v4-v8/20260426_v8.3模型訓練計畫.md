# BinauralTFNet v8.3 訓練計畫

## 實驗目標

修正 v8/v8.2 的根本問題：**從 y_common 出發，而不是從 warped 出發**，讓 SpecificBranch 有完全的控制權。

---

## v8/v8.2 失敗原因回顧

### 共同問題：從 warped 出發的設計缺陷

```python
# v8/v8.2 的設計
Y_L = Y_warp_L + delta_L  # 從 Warpnet 的輸出出發
Y_R = Y_warp_R + delta_R
```

**假設**：Warpnet 的 ITD 是對的，SpecificBranch 只需要修正細節

**實際**：
- Warpnet 的 ITD 誤差很大（v8 測試 772μs）
- SpecificBranch 只加 delta 無法完全修正基礎錯誤
- 導致 Stage 2 改善不足（v8: 13%, v8.2: 8.5%）

### v8.2 的修改沒有解決根本問題

v8.2 修改：
- DPAB 分離位置/方向 ✓
- 增大 STFT 解析度 ✓
- 增大模型容量 ✓

**結果**：Stage 2 改善反而更差（8.5% vs v8 的 13%）

**結論**：問題不在 DPAB 實作或模型容量，而在**從 warped 出發的架構設計**。

---

## v8.3 核心修改

### 1. **修正 CommonBranch（關鍵發現）**

**問題**：v8/v8.2 的 CommonBranch 把 view 傳給 HyperConvWavenet
```python
# 錯誤：y_common 會受到 view（角度）影響
_, skips = self.hyperconv_wavenet(x, view)
y_common = self.output_net(skips)
```

**結果**：
- y_common 不是真正的「共同部分」
- y_common 會隨著角度變化
- SpecificBranch 無法學到「差異」

**修正**：用固定的 view_front（正前方）
```python
# 創建固定的 view（正前方）
view_front = th.zeros_like(view)
view_front[:, 1, :] = -1.0  # y = -1（正前方）
view_front[:, 3, :] = 1.0   # quat_w = 1

_, skips = self.hyperconv_wavenet(x, view_front)  # 用固定的 view
y_common = self.output_net(skips)
```

**這可能是 v8/v8.2 失敗的根本原因！**

### 2. **從 y_common 出發（不是 warped）**

```python
# v8/v8.2（錯誤）
warped_L, warped_R = Warpnet(mono, view)
Y_warp_L = STFT(warped_L)
Y_warp_R = STFT(warped_R)
Y_L = Y_warp_L + delta_L  # 依賴 Warpnet

# v8.3（修正）
y_common = WaveNet(mono, view)  # CommonBranch 輸出
Y_common = STFT(y_common)       # 對共同部分做 STFT
Y_L = Y_common + delta_L        # SpecificBranch 負責全部的左右差異
Y_R = Y_common + delta_R
```

**優點**：
- SpecificBranch 有完全的控制權
- 不依賴 Warpnet 的 ITD 準確度
- delta 可以學習完整的空間資訊（ITD + ILD + HRTF）

**缺點**：
- SpecificBranch 的任務更重（要學 ITD）
- 可能需要更多訓練時間

### 2. **簡化 DPAB（移除插值）**

```python
class SimpleDPAB(nn.Module):
    """
    簡化版 DPAB，不做插值
    用 Conv1d 直接處理 view (B×7×K) → (B×cond_dim×T_stft)
    """
    def __init__(self, view_dim=7, cond_dim=64):
        super().__init__()
        self.conv1 = nn.Conv1d(view_dim, cond_dim, kernel_size=3, padding=1)
        self.conv2 = nn.Conv1d(cond_dim, cond_dim, kernel_size=3, padding=1)
        self.conv3 = nn.Conv1d(cond_dim, cond_dim, kernel_size=3, padding=1)
    
    def forward(self, view, t_stft):
        # view: B×7×K
        x = F.relu(self.conv1(view))      # B×cond_dim×K
        x = F.relu(self.conv2(x))         # B×cond_dim×K
        x = self.conv3(x)                 # B×cond_dim×K
        
        # 插值到 T_stft
        x = F.interpolate(x, size=t_stft, mode='linear', align_corners=False)
        return x  # B×cond_dim×T_stft
```

**改進**：
- 移除 Cross-attention（可能沒有學到東西）
- 用簡單的 Conv1d 處理 view
- 減少參數量，加快訓練

### 3. **調整訓練階段**

```python
# v8/v8.2
Stage 1: 0-80   (CommonBranch)
Stage 2: 80-160 (SpecificBranch)
Stage 3: 160-180 (全部)

# v8.3
Stage 1: 0-60   (CommonBranch，減少 20 epochs)
Stage 2: 60-160 (SpecificBranch，增加 20 epochs)
Stage 3: 160-180 (全部)
```

**理由**：
- Stage 1 的 L2 在 60 epochs 就收斂得很好了
- Stage 2 的任務更重（要學 ITD），需要更多時間

## 代碼修改

### 1. 修改 SpecificBranch.forward

```python
# 舊版（v8/v8.2）
def forward(self, warped, view, seq_length):
    warped_L = warped[:, 0:1, :]
    warped_R = warped[:, 1:2, :]
    Y_warp_L = self._stft(warped_L)
    Y_warp_R = self._stft(warped_R)
    
    cond = self.dpab(view, T_stft)
    delta_L, delta_R = self.tf_resstack(Y_warp_L, Y_warp_R, cond)
    
    Y_L = Y_warp_L + delta_L  # 從 warped 出發
    Y_R = Y_warp_R + delta_R

# 新版（v8.3）
def forward(self, y_common, view, seq_length):
    Y_common = self._stft(y_common)  # B×1×T → B×F×T_stft×2
    
    # 複製成左右聲道
    Y_common_L = Y_common
    Y_common_R = Y_common
    
    cond = self.dpab(view, T_stft)
    delta_L, delta_R = self.tf_resstack(Y_common_L, Y_common_R, cond)
    
    Y_L = Y_common + delta_L  # 從 y_common 出發
    Y_R = Y_common + delta_R
    
    y_L = self._istft(Y_L, seq_length)
    y_R = self._istft(Y_R, seq_length)
    return th.cat([y_L, y_R], dim=1)
```

### 2. 修改 BinauralTFNet.forward

# 舊版
def forward(self, mono, view):

## 風險評估

### 風險 1：SpecificBranch 任務太重

**原因**：
- 現在要學習完整的 ITD + ILD + HRTF
- 之前只需要學習「細節修正」

**緩解**：
- 增加 Stage 2 訓練時間（80 → 100 epochs）
- 保持 v8.2 的大模型容量
- 監控 ITD loss（雖然不參與訓練，但可以觀察）

### 風險 2：SimpleDPAB 太簡單

**原因**：
- 移除了 Cross-attention
- 可能無法捕捉位置和方向的交互作用

**緩解**：
- 先試試看，如果不行再加回 attention
- 或者用 Transformer 替代 Conv1d

### 風險 3：從 y_common 出發可能破壞 ITD

**原因**：
- y_common 是左右平均，沒有 ITD 資訊
- SpecificBranch 要從零開始學 ITD

**緩解**：
- 這正是我們想要的！讓 SpecificBranch 完全控制
- 如果 Stage 2 學不到 ITD，考慮在 Stage 2 加入 ITD loss

## 備註

如果 v8.3 仍然失敗（Phase 改善 <25%），考慮：

1. **在 Stage 2 加入 ITD loss**：
   - 幫助 SpecificBranch 學習 ITD
   - 但要小心 loss 量級平衡

2. **改用 Transformer 替代 Conv1d**：
   - 更強的序列建模能力
   - 但參數量和訓練時間增加

3. **完全放棄 CommonBranch**：
   - 直接用 SpecificBranch 從 mono 合成左右耳
   - 類似 DPATFNet 的純頻域方法
