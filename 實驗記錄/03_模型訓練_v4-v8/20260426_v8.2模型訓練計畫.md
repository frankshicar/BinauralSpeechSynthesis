# BinauralTFNet v8.2 訓練計畫

## 實驗目標

修正 v8 的 DPAB 實作錯誤，增大模型容量，目標達到 DPATFNet 水準（Phase ~0.70）。

## v8 失敗原因回顧

1. **DPAB 實作錯誤（主因）**：把整個 view (7 維) 同時投影成 P 和 O，導致 Cross-attention 語義混亂
2. **STFT 解析度不足**：fft_size=512 太小，頻率解析度不夠
3. **TFResStack 容量不足**：4 blocks / 128 channels 太淺

**結果**：Stage 2 改善只有 13%（應該 50%+），Phase 1.57（目標 0.70）

---

## v8.2 修改內容

### 1. **修正 DPAB（關鍵修改）**

#### 問題診斷

v8 的錯誤實作：
```python
# 把全部 7 維同時當作位置和方向
v = view.transpose(1, 2)  # B×K×7
pos = self.proj_pos(v)    # Linear(7 → 64)
dir_ = self.proj_dir(v)   # Linear(7 → 64)
cond1, _ = self.cross_attn(pos, dir_, dir_)
```

這導致：
- P 和 O 都包含 [x, y, z, quat_w, quat_x, quat_y, quat_z]
- Cross-attention 的 Q/K/V 語義混亂
- 無法學習位置和方向的交互作用

#### 正確實作

```python
class DPAB(nn.Module):
    """
    Dual Position Attention Block
    分離位置 (P) 和方向 (O)，用 Cross-attention 融合
    """
    def __init__(self, pos_dim=3, ori_dim=4, cond_dim=64, nhead=4):
        super().__init__()
        self.proj_pos = nn.Linear(pos_dim, cond_dim)  # 3 → 64
        self.proj_ori = nn.Linear(ori_dim, cond_dim)  # 4 → 64
        self.cross_attn = nn.MultiheadAttention(cond_dim, nhead, batch_first=True)
        self.self_attn  = nn.MultiheadAttention(cond_dim, nhead, batch_first=True)
        self.out_proj   = nn.Conv1d(cond_dim, cond_dim, kernel_size=1)

    def forward(self, view, t_stft):
        # view: B×7×K → 分離位置和方向
        pos = view[:, 0:3, :].transpose(1, 2)  # B×K×3 [x, y, z]
        ori = view[:, 3:7, :].transpose(1, 2)  # B×K×4 [quat_w, quat_x, quat_y, quat_z]
        
        P = self.proj_pos(pos)  # B×K×cond_dim
        O = self.proj_ori(ori)  # B×K×cond_dim
        
        # PAL1: P 作 Query，O 作 Key/Value
        cond1, _ = self.cross_attn(P, O, O)
        cond1 = cond1 + P  # Residual connection
        
        # PAL2: Self-attention（捕捉位置變化 / Doppler）
        cond2, _ = self.self_attn(cond1, cond1, cond1)
        
        # 插值到 T_stft
        cond = cond2.transpose(1, 2)  # B×cond_dim×K
        cond = F.interpolate(cond, size=t_stft, mode='linear', align_corners=False)
        return self.out_proj(cond)  # B×cond_dim×T_stft
```

**改進點**：
- 明確分離 pos (3 維) 和 ori (4 維)
- Cross-attention 語義清晰：位置 query 方向
- 加入 Residual connection（DPATFNet 的設計）

### 2. **增大 STFT 解析度**


## 風險評估與緩解

### 風險 1：記憶體不足

**原因**：
- F: 257 → 513（2 倍）
- tf_channels: 128 → 256（2 倍）
- 總記憶體增加 ~4 倍

**緩解**：
- batch_size: 32 → 16
- 如果還不夠，降到 8
- 監控 GPU 記憶體使用率

### 風險 2：訓練時間過長

**原因**：
- 模型變大 + batch_size 減半
- 預計 24-30 小時

**緩解**：
- 可接受，如果結果顯著改善
- 如果 epoch 80 後 Phase 沒有明顯下降（<1.0），立即停止分析

### 風險 3：DPAB 修正後可能需要調整 learning rate

**原因**：
- 新的 DPAB 結構可能需要不同的 lr
- Cross-attention 的梯度特性不同

**緩解**：
- 監控 Stage 2 的 loss 下降速度
- 如果前 10 epochs 沒有下降，考慮提高 lr 到 5e-4
- 或者只提高 SpecificBranch 的 lr

### 風險 4：從 warped 出發的設計仍然有問題

**原因**：
- 如果 Warpnet 的 ITD 本身就很差
- SpecificBranch 加 delta 可能無法完全修正

**緩解**：
- 先跑 v8.2，看 ITD 是否改善
- 如果 ITD 仍然 >500μs，考慮 v8.3：
  - 改成從 y_common 出發：`Y_L = Y_common + delta_L`
  - 或者增大 delta 的權重（Tanh → 2×Tanh）

### 訓練時間

### Stage 2 結果（epoch 80-108）❌

```

IPD: 1.250 → 1.035（改善 17.2%）← 目標 40%+

```

**對比 v8**：
- v8 Phase 改善：13%
- v8.2 Phase 改善：8.5%
- **v8.2 甚至更差**

### 失敗原因分析

#### 1. **DPAB 修正沒有解決根本問題**

雖然分離了位置 (x,y,z) 和方向 (quat)，但 Stage 2 改善幅度反而更差（8.5% vs v8 的 13%）。

**可能原因**：
- 插值丟失資訊（K=24 → T_stft=75）
- Cross-attention 沒有有效融合位置和方向
- Self-attention 沒有捕捉到位置變化

#### 2. **從 warped 出發的設計是根本問題**

```python
Y_L = Y_warp_L + delta_L  # 從 Warpnet 的輸出出發
```

**問題**：
- 假設 Warpnet 的 ITD 是對的
- 但實際上 Warpnet 的 ITD 誤差很大（v8 測試 772μs）
- SpecificBranch 只加 delta 無法完全修正基礎錯誤

**證據**：
- v8 和 v8.2 的 Stage 2 都改善不足
- 說明問題不在 DPAB 的實作細節
- 而在於整體架構設計

#### 3. **增大模型容量沒有幫助**

v8.2 的修改：
- fft_size: 512 → 1024
- tf_channels: 128 → 256
- tf_blocks: 4 → 8
- 參數量：9.06M → 11.74M (+30%)

**結果**：Stage 2 改善反而更差（8.5% vs 13%）

**結論**：問題不在模型容量，而在架構設計。

## v8.3 修改方向

### 核心修改：從 y_common 出發

```python
# v8/v8.2（錯誤）
Y_L = Y_warp_L + delta_L  # 依賴 Warpnet 的 ITD

# v8.3（修正）
Y_common = STFT(y_common)  # 從 CommonBranch 的輸出出發
Y_L = Y_common + delta_L   # SpecificBranch 負責全部的左右差異
Y_R = Y_common + delta_R
```

**優點**：
- SpecificBranch 有完全的控制權
- 不依賴 Warpnet 的 ITD 準確度
- delta 可以學習完整的空間資訊（ITD + ILD + HRTF）

**缺點**：
- SpecificBranch 的任務更重（要學 ITD）
- 可能需要更大的模型容量

### 次要修改

1. **移除 DPAB 的插值**：
   - 用 Conv1d 處理 view (B×7×K)
   - 直接輸出 B×cond_dim×T_stft（不插值）

2. **增加 Stage 2 訓練時間**：
   - Stage 1: 0-60（減少 20 epochs）
   - Stage 2: 60-160（增加 20 epochs）
   - Stage 3: 160-180

3. **保持 v8.2 的模型容量**：
   - fft_size=1024, tf_channels=256, tf_blocks=8
