# 時間對齊修正計劃 (Temporal Alignment Fix)

## 問題分析 (Problem Analysis)

### 當前評估結果

| 指標 | 你的結果 | 預訓練模型基準 | 狀態 |
|------|---------|--------------|------|
| L2 × 10³ | **0.040** | 0.144 | ✅ **比基準好 3.6 倍** |
| Amplitude | **0.017** | 0.036 | ✅ **比基準好 2.1 倍** |
| Phase (rad) | **1.570** | 0.804 | ❌ **比基準差 1.95 倍** |

### 根本原因

**Phase error = 1.57 弧度 ≈ 90° 極度異常！**

可能的原因：
1. ✅ **時間對齊問題** (最可能) - mono.wav 和 binaural.wav 起始時間不一致
2. ⚠️ HRTF 不匹配 - 測試假人頭與訓練假人頭不同
3. ⚠️ 座標系統錯誤 - 已修正

### 為什麼時間對齊如此關鍵？

**Phase 對時間偏移極度敏感**：

```
時間延遲 Δt = 0.25 ms 時：
- 對於 1000 Hz 的聲音 → 相位偏移 = 2π × 1000 × 0.00025 ≈ 1.57 rad ≈ 90°
- 這正好等於你的 phase error！
```

在 48kHz 採樣率下：
- **1 個 sample 的延遲** = 1/48000 ≈ 0.021 ms
- 對於 1000 Hz → 相位偏移 ≈ 0.13 rad ≈ 7.5°
- **12 個 samples 的延遲** → **90° 相位偏移**

## 解決方案 (Proposed Solution)

### 方法：Cross-Correlation Alignment

使用互相關 (Cross-Correlation) 自動偵測並修正 mono 和 binaural 之間的時間偏移。

#### 原理

1. **計算互相關**：比較 mono 信號與 binaural 左聲道的相似度
2. **找到最大值**：最大相關性的位置 = 最佳對齊點
3. **對齊信號**：根據偏移量調整音訊起始位置

### 實作步驟

#### 1. 創建對齊工具函數

新增 `src/alignment.py`：
- `find_alignment_offset()` - 使用 cross-correlation 找到偏移量
- `align_signals()` - 對齊兩個音訊信號
- `diagnose_alignment()` - 輸出診斷資訊

#### 2. 修改評估腳本

在 [evaluate.py](file:///home/sbplab/frank/BinauralSpeechSynthesis/evaluate.py) 中加入對齊步驟：

**當前流程**：
```python
mono = load("mono.wav")
binaural = load("binaural.wav")
predicted = model(mono)
error = compute_metrics(predicted, binaural)  # ❌ 未對齊
```

**修正後流程**：
```python
mono = load("mono.wav")
binaural_ref = load("binaural.wav")

# ✅ 新增：時間對齊
offset = find_alignment_offset(mono, binaural_ref)
mono, binaural_ref = align_signals(mono, binaural_ref, offset)

predicted = model(mono)
error = compute_metrics(predicted, binaural_ref)  # ✅ 已對齊
```

#### 3. 診斷輸出

為每個 subject 輸出：
- 偵測到的時間偏移（samples 和 ms）
- 對齊前後的相關性
- 對齊是否成功

## 額外考量 (Additional Considerations)

### 如果對齊後 Phase 仍然很高

可能需要：
1. **使用相同假人頭重新錄製測試數據**
2. **用測試數據重新訓練模型**
3. **實作 HRTF 校準/轉換**

### 對齊的副作用

- 音訊長度可能略微改變
- 需要確保位置數據 (tx_positions.txt) 也對應調整

### 訓練數據是否需要對齊？

**Facebook 原始數據集可能已經對齊**，所以訓練時不需要。但你的測試數據可能是自己錄製的，所以需要對齊。
