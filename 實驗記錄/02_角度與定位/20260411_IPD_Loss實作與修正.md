## 一、實作摘要

為了解決模型 ITD 系統性偏大和左右不對稱的問題，我們加入了 IPD (Interaural Phase Difference) Loss 來直接約束雙耳相位差。

### 修改的檔案

1. **src/losses.py** - 新增 `IPDLoss` 類別
2. **src/trainer.py** - 整合 IPD loss 到訓練流程
3. **train.py** - 設定 IPD loss 權重

### 核心功能

**IPDLoss 類別**：
- 在頻域直接計算左右耳的相位差（IPD）
- 比時域的 ITD 更精確地約束雙耳時間差
- 使用能量遮罩忽略低能量頻率成分，提高數值穩定性
- 正確處理相位的週期性（wrap to [-π, π]）

## 三、發現的問題（2026-04-12）

### 訓練記錄分析

從訓練記錄中發現了幾個嚴重問題：

#### 1. IPD Loss 沒有下降 ❌
```
Epoch 1:  ipd = 1.517
Epoch 2:  ipd = 1.569
Epoch 3:  ipd = 1.570
...
Epoch 13: ipd = 1.569
```
IPD loss 從第二個 epoch 開始就幾乎沒有變化，模型完全沒有在學習 IPD。

#### 2. Learning Rate 下降太快 ❌
```
Epoch 1:  lr = 0.001
Epoch 2:  lr = 0.0005
Epoch 3:  lr = 0.00025
...
Epoch 8:  lr = 0.0000078
Epoch 9-13: lr = 0.0000078  ← 卡住了
```
8 個 epoch 內 learning rate 就掉了 128 倍，然後卡在最小值。

#### 3. Phase Loss 也是常數 ❌
所有 epoch 的 phase loss ≈ 1.569，正常訓練應該要持續下降。

### 根本原因

#### IPD Loss 的問題

1. **形狀不匹配**：原始實作中 STFT 輸出形狀處理錯誤
2. **能量遮罩過濾太多**：過濾掉太多頻率，導致梯度無法傳播
3. **缺少 else 分支**：當沒有有效頻率時返回 None，導致梯度中斷

#### Learning Rate 的問題

`newbob_max_decay: 0.01` 太激進，允許 learning rate 衰減到原來的 1%。

## 四、修正方案（2026-04-12）

### 1. 重寫 IPD Loss

參考 PhaseLoss 的實作方式：

```python
class IPDLoss(Loss):
    def _loss(self, data, target):
        # 對左右聲道分別做 STFT，然後 flatten
        pred_left_stft = self.fft.stft(data[:, 0, :]).view(-1, 2)
        pred_right_stft = self.fft.stft(data[:, 1, :]).view(-1, 2)
        target_left_stft = self.fft.stft(target[:, 0, :]).view(-1, 2)
        target_right_stft = self.fft.stft(target[:, 1, :]).view(-1, 2)
        
        # 能量遮罩（與 PhaseLoss 相同的方式）
        target_mask = (target_left_energy > threshold) & (target_right_energy > threshold)
        pred_mask = (pred_left_energy > threshold) & (pred_right_energy > threshold)
        indices = th.nonzero(target_mask * pred_mask).view(-1)
        
        # 如果有效頻率太少，使用所有頻率
        if indices.numel() < 10:
            indices = th.arange(pred_left_stft.shape[0], device=data.device)
        
        # 計算 IPD 並計算誤差（與 PhaseLoss 相同的週期性處理）
        pred_ipd = pred_left_phase - pred_right_phase
        target_ipd = target_left_phase - target_right_phase
        ipd_diff = th.abs(pred_ipd - target_ipd)
        ipd_diff = np.pi - th.abs(ipd_diff - np.pi)
        
        return th.mean(ipd_diff)
```

**關鍵改進：**
- ✅ 正確處理 STFT 輸出形狀
- ✅ 使用與 PhaseLoss 相同的能量遮罩方式
- ✅ 確保總是返回有效的 loss 值
- ✅ 使用與 PhaseLoss 相同的週期性處理

## 五、技術細節

### IPD vs ITD

- **ITD (時域)**：計算左右耳訊號的時間延遲，使用互相關方法
- **IPD (頻域)**：計算每個頻率的相位差，更精確且可微分

### 為什麼 IPD 更有效？

1. **頻率選擇性**：不同頻率的 ITD 可以分別約束
2. **數值穩定**：避免時域互相關的離散化誤差
3. **梯度傳播**：頻域計算對神經網路更友好
4. **物理約束**：直接對應人耳的相位鎖定機制（< 1500 Hz）

### 能量遮罩的作用

```python
energy_threshold = self.ignore_below * th.mean(target_energy, dim=-1, keepdim=True)
mask = (target_energy > energy_threshold) & (pred_energy > energy_threshold)
```
- 忽略低能量頻率（通常是噪音）
- 提高數值穩定性
- 避免在無意義的頻率上懲罰模型

## 六、預期效果

### 解決的問題

1. **ITD 系統性偏大**：IPD loss 直接約束頻域相位差，防止 ITD 超出物理極限（±800μs）

## 十、故障排除

### 問題：訓練時 ipd loss 不下降

**可能原因：**
- IPD 權重太小，模型忽略了這個 loss
- 學習率太低

**解決方法：**
- 增加 ipd 權重到 0.1 或 0.15
- 檢查其他 loss 是否正常下降

### 問題：聲音品質明顯下降

**可能原因：**
- IPD 權重太大，犧牲了波形重建品質

**解決方法：**
- 降低 ipd 權重到 0.03
- 增加訓練 epoch 數，讓模型有更多時間平衡各個 loss

### 問題：左右不對稱仍然存在

**可能原因：**
- 訓練資料本身不對稱
- 模型架構的偏差

**解決方法：**
- 檢查訓練資料的角度分布
- 考慮資料增強（左右翻轉）
- 增加 IPD 權重
