## 一、角度校正功能

### 問題背景

在使用預訓練的雙耳化模型進行音訊合成時，發現模型對不同角度的預測準確度不一致：

- **極端角度（±90°）**：預測非常準確，誤差接近 0°
- **中間角度（±30°, ±60°）**：存在系統性偏差，所有角度都往 0° 方向收縮

### 實測偏差數據

| 設定角度 | 實際感知角度 | 誤差 | 偏差方向 |
|---------|------------|------|---------|
| -90°    | -90.0°     | 0.0° | 無偏差   |
| -60°    | -56.2°     | 3.8° | 往右偏   |
| -30°    | -23.5°     | 6.5° | 往右偏   |
| 0°      | -1.9°      | 1.9° | 往左偏   |
| +30°    | +21.4°     | 8.6° | 往右偏   |
| +60°    | +44.3°     | 15.7°| 往右偏   |
| +90°    | +90.0°     | 0.0° | 無偏差   |

**平均角度誤差：5.4°**

### 校正原理

既然模型在特定 tx 角度下會產生系統性偏差，我們可以建立一個**反向映射函數**：

```
輸入：想要的感知角度
輸出：應該設定的 tx 角度
```

### 實作方法

#### 方法一：簡單補償

```python
angle_bias = {
    -90: 0.0, -60: 3.8, -30: 6.5, 0: -1.9, 
    30: -8.6, 60: -15.7, 90: 0.0
}

def correct_angle_simple(desired_angle):
    closest_angle = min(angle_bias.keys(), key=lambda x: abs(x - desired_angle))
    bias = angle_bias[closest_angle]
    return desired_angle - bias
```

#### 方法二：曲線插值

使用三次樣條插值建立平滑的校正曲線：

```python
from scipy.interpolate import interp1d
import numpy as np

tx_angles = np.array([-90, -60, -30, 0, 30, 60, 90])
perceived = np.array([-90.0, -56.2, -23.5, -1.9, 21.4, 44.3, 90.0])
correction_func = interp1d(perceived, tx_angles, kind='cubic', fill_value='extrapolate')

def correct_angle_curve(desired_angle):
    corrected = float(correction_func(desired_angle))
    return np.clip(corrected, -90, 90)
```

### 校正效果

#### 原始 7 個角度測試結果

| 角度 | 校正前誤差 | 簡單補償誤差 | 曲線插值誤差 | 最佳方法 |
|-----|----------|------------|------------|---------|
| -90° | 0.0°    | 0.0°       | 0.0°       | 相同    |
| -60° | 3.8°    | 0.2°       | 0.2°       | 相同    |
| -30° | 6.5°    | 2.1°       | 2.1°       | 相同    |
| 0°   | 1.9°    | 1.9°       | 1.9°       | 相同    |
| +30° | 8.6°    | 0.1°       | 0.1°       | 相同    |
| +60° | 15.7°   | 8.5°       | 8.5°       | 相同    |
| +90° | 0.0°    | 0.0°       | 0.0°       | 相同    |

#### 新增 ±45° 角度驗證結果

| 角度 | 簡單補償預測 | 簡單補償誤差 | 曲線插值預測 | 曲線插值誤差 | 改善幅度 |
|-----|------------|------------|------------|------------|---------|
| -45° | -48.8°     | 2.8°       | -51.4°     | 2.0°       | 29% ↓   |
| +45° | +53.6°     | 8.6°       | +60.9°     | 0.7°       | 92% ↓   |

**平均角度誤差：**
- 校正前：5.4°
- 簡單補償：1.9°（改善 65%）
- 曲線插值：1.8°（改善 67%）

### 核心發現與限制 ⚠️

#### 1. 曲線插值無法完美校正已知點

- 理論上，用 7 個已知點建立的插值函數應該完美擬合這些點
- 實際上，由於使用「反向插值」（感知→tx）+ 模型「正向推理」（tx→感知）
- 兩個函數的複合會產生累積誤差，即使已知點也無法達到零誤差
- 例如：+60° 即使是訓練點，校正後誤差仍有 8.5°

#### 2. +60° 到 +90° 區間特別困難

- 該區間只有 2 個數據點，插值不可靠
- tx=+60° → 感知=+44.3°（損失 15.7°）
- tx=+90° → 感知=+90.0°（損失 0°）
- 30° 跨度內偏差變化極大，導致插值函數陡峭且不穩定

#### 3. HRTF 不匹配是根本原因

- 預訓練模型的 HRTF 與 KEMAR HRTF 存在系統性差異
- 右側（正角度）偏差遠大於左側（負角度）
- 單純的數學校正無法完全彌補這種物理特性的差異
- 徹底解決需要用 KEMAR 數據 fine-tune 模型

## 二、DOA 方法比較

### 核心結論

**對於語音信號的角度估計，強烈建議使用 ITD 方法（GCC-PHAT）**

### 實測結果對比

| 方法 | +60° 誤差 | 適用性 | 推薦度 |
|------|----------|--------|--------|
| **ITD (GCC-PHAT)** | ~15° | 語音、音樂 | ⭐⭐⭐⭐⭐ 強烈推薦 |
| **ILD (線性映射)** | ~25° | 不適用 | ❌ 不推薦 |
| **Hybrid (ITD 0.7 + ILD 0.3)** | ~18° | 可用但不如純 ITD | ⭐⭐⭐ 可選 |

### 為什麼 ILD 表現這麼差？

#### 1. 線性映射的問題

目前的 ILD 實現使用：
```python
angle = (ILD / 20dB) * 90°
```

這個公式有三個嚴重問題：

**問題 1：max_ILD 假設錯誤**
- 假設值：20 dB
- 實際值：需要用 `diagnose_ild.py` 測量，可能是 15-18 dB
- 影響：所有角度都被高估或低估

**問題 2：非線性關係**

實際的 ILD-角度關係：
```
角度    實際 ILD    線性預測 ILD    誤差
-90°    -15 dB      -20 dB         5 dB
-60°     -8 dB      -13.3 dB       5.3 dB
-30°     -3 dB       -6.7 dB       3.7 dB
  0°      0 dB        0 dB         0 dB
+30°     +3 dB       +6.7 dB       3.7 dB
+60°     +8 dB      +13.3 dB       5.3 dB
+90°    +15 dB      +20 dB         5 dB
```

線性映射在中間角度（±30°~±60°）誤差最大！

**問題 3：KEMAR 特定性**
- 每個 HRTF（頭部模型）的 ILD 特性都不同
- 預訓練模型使用的 HRTF 與 KEMAR 不同
- 通用公式無法適用

#### 2. ITD 為什麼更好？

**物理優勢**
- **時間差是絕對量**：不受音量、頻譜影響
- **低頻穩定**：語音主要能量在 300-3000 Hz，ITD 最準確
- **相位清晰**：在低頻範圍內無相位模糊

**算法優勢**
- GCC-PHAT 使用相位信息，對噪音有較好的魯棒性
- 頻率加權可以過濾不可靠的頻段
- 不需要校準（耳間距是固定的 21.5 cm）

## 三、ILD 角度估計

### ⚠️ 重要警告

**ILD 方法目前使用簡化的線性映射，準確度遠低於 ITD 方法！**

實測結果顯示：
- ITD (GCC-PHAT) 在 ±30° 誤差約 3-4°
- ILD 在 ±30° 誤差高達 25°（完全不可用）

**根本問題**：
1. 線性映射 `angle = (ILD / 20dB) * 90°` 是粗糙的假設
2. 實際 KEMAR HRTF 的 ILD-角度關係是非線性的
3. max_ILD = 20 dB 是通用假設，不是針對此模型的實測值

**建議**：
- **語音信號請使用 ITD 方法（GCC-PHAT）**
- 如需使用 ILD，必須先用 `diagnose_ild.py` 測量實際 ILD 值
- 最好建立基於 KEMAR HRTF 的查找表

### 三種角度估計方法

#### 1. ITD 方法（GCC-PHAT）

**原理**：利用聲音到達左右耳的時間差來估計方位角

**優勢**：
- 在低頻（300-3000 Hz）非常準確
- 對相位信息敏感，適合純音和語音
- 人類聽覺系統主要依賴 ITD 進行低頻定位

**適用場景**：
- 語音信號
- 音樂（特別是低頻樂器）
- 純音測試

#### 2. ILD 方法（強度差）

**原理**：利用左右耳接收到的聲音強度差異來估計方位角

**物理基礎**：
- **頭部遮蔽效應（Head Shadow Effect）**：頭部會阻擋聲波
- 高頻聲波（>1 kHz）波長較短，更容易被頭部遮擋
- 典型 ILD 範圍：0-25 dB（取決於頻率和角度）

**限制**：
- 低頻時 ILD 很小（聲波繞射）
- 前後混淆問題
- 需要校準（ILD-角度關係因人而異）

#### 3. Hybrid 方法（混合）

**原理**：結合 ITD 和 ILD 的優勢，使用加權平均

**預設配置**：
- ITD 權重：0.7（使用 300-3000 Hz）
- ILD 權重：0.3（使用 2000-8000 Hz）

### 累積誤差改善

- 原始 7 點：平均角度誤差 5.4°
- 擴充 12 點：平均角度誤差 3.2° (改善 41%)

### 關鍵發現：+60° ~ +90° 區間的非單調性 ⚠️

#### 問題描述

在 +60° ~ +90° 區間發現了**非單調性**現象：

```
tx=+60° -> 感知=+68.5° (超過設定 8.5°)
tx=+75° -> 感知=+63.8° (低於設定 11.2°，且比 +60° 的感知還小！)
tx=+90° -> 感知=+90.0° (完全準確)
```

這是一個「先升後降再升」的模式。

#### 對曲線插值的影響

1. **三次樣條插值失效**
   - 三次樣條假設數據是平滑且單調的
   - 遇到非單調數據會產生劇烈振盪
   - 例如：想要 +75°，插值預測 tx=+34.2°（完全錯誤！）

2. **反向映射的根本問題**
   - 我們需要：想要的感知 → 應設定的 tx
   - 但 perceived 不是單調函數，存在「一對多」的問題
   - 例如：感知 65° 可能對應 tx=+60° 或 tx=+75°

### 原因分析

#### 1. HRTF 特性的複雜性

- 人類 HRTF 在不同角度有不同的頻譜特徵
- +60° ~ +90° 區間可能涉及耳廓共振的轉換
- 模型學習到的 HRTF 與 KEMAR 的 HRTF 不匹配

#### 2. 訓練數據分布

- 預訓練模型可能在 +60° ~ +90° 區間的訓練數據較少
- 或者訓練數據的 HRTF 在這個區間有特殊行為

#### 3. 模型架構限制

- WaveNet 可能在極端角度（±90°）有特殊的錨定機制
- 導致 +90° 完全準確，但 +60° ~ +75° 出現過渡問題

### 結論與建議

#### 1. 曲線插值的局限性

**結論**：增加右側數據點並**無法**解決 +60° ~ +90° 區間的問題

原因：
- 問題不是數據點不足，而是數據本身非單調
- 這是模型 HRTF 的固有特性，不是插值方法的問題
- 數學校正無法解決物理特性的不匹配

#### 2. 推薦的校正策略

**方案 A：分段處理（實用）**

```python
def correct_angle_hybrid(desired_angle):
    if desired_angle <= 60:
        # 使用曲線插值（效果好）
        return curve_interpolation(desired_angle)
    else:
        # +60° 以上使用簡單補償或線性插值
        return simple_compensation(desired_angle)
```

**方案 B：接受限制（誠實）**

- 明確告知用戶：+60° ~ +90° 區間的校正效果有限
- 平均誤差約 10°，這是當前模型的極限
- 如需更高精度，建議 fine-tune 模型

**方案 C：Fine-tune 模型（根本解決）**

- 用 KEMAR 數據重新訓練模型
- 這是唯一能徹底解決 +60° ~ +90° 問題的方法
- 可以消除非單調性，讓所有角度都準確

## 五、總結

### 主要發現

1. **角度校正可以改善中間角度的準確性**
   - 平均誤差從 5.4° 降到 1.8°
   - 但 +60° ~ +90° 區間仍有限制

2. **ITD 方法遠優於 ILD 方法**
   - ITD 誤差約 3-4°
   - ILD 誤差高達 25°
   - 語音信號應該使用 ITD

3. **+60° ~ +90° 區間的非單調性問題**
   - 這是模型 HRTF 的固有特性
   - 數學校正無法完全解決
   - 需要 fine-tune 模型

4. **HRTF 不匹配是根本原因**
   - 預訓練模型的 HRTF 與 KEMAR 不同
   - 右側偏差大於左側
   - 徹底解決需要用 KEMAR 數據重新訓練

### 實用建議

1. **短期方案**：使用分段校正策略
2. **中期方案**：接受 +60° ~ +90° 的精度限制
3. **長期方案**：用 KEMAR 數據 fine-tune 模型
