## 一、Testset 設計說明

### 為什麼所有 subject 的 mono.wav 都相同？

這是標準的 HRTF 測試設計，**完全正常且合理**。

#### 測試邏輯

```
相同的音源 (mono.wav)
    ↓
不同的 HRTF (subject1, subject2, ...)
    ↓
不同的雙耳輸出 (binaural.wav)
```

#### 檔案結構

```
dataset/testset_org/
├── subject1/
│   ├── mono.wav          # 相同的測試信號
│   ├── binaural.wav      # subject1 的真實雙耳錄音
│   ├── tx_positions.txt  # subject1 的頭部軌跡
│   └── rx_positions.txt
├── subject2/
│   ├── mono.wav          # 相同的測試信號（與 subject1 相同）
│   ├── binaural.wav      # subject2 的真實雙耳錄音（不同）
│   └── tx_positions.txt  # subject2 的頭部軌跡（不同）
...
```

#### 驗證

```bash
$ md5sum dataset/testset_org/subject*/mono.wav
a17eaa56fdb1b83dcefe37141d8c1c8b  subject1/mono.wav
a17eaa56fdb1b83dcefe37141d8c1c8b  subject2/mono.wav
a17eaa56fdb1b83dcefe37141d8c1c8b  subject3/mono.wav
# 所有 mono.wav 的 MD5 相同 ✓
```

### 為什麼 calibrate_angle.py 結果相同？

#### 輸入分析

```bash
python calibrate_angle.py --input dataset/testset_org/subject1/mono.wav --desired 60
python calibrate_angle.py --input dataset/testset_org/subject2/mono.wav --desired 60
```

**輸入完全相同**：
1. mono.wav：相同（MD5 相同）
2. 模型：相同（同一個 .net 檔案）
3. tx 角度範圍：相同（auto 掃描 48°~72°）

**輸出必然相同**：
- 相同的輸入 → 相同的模型 → 相同的輸出
- 這是確定性系統的正常行為

#### 結果解讀

```
最佳: tx = +70.00° -> 估計 +56.19° | 誤差 3.81°
```

這個結果的含義：
- 對於**這個特定的 mono.wav**
- 使用**這個預訓練模型**
- 想要感知 +60°，需要設定 tx = +70°
- 實際估計會是 +56.19°（還差 3.81°）

### 正確的測試方法

#### 1. 評估模型整體表現（推薦）

```bash
python evaluate.py \
    --dataset_directory ./dataset/testset_org \
    --model_file outputs/binaural_network.newbob.net \
    --artifacts_directory results_audio
```

這會：
- 對每個 subject 讀取其 `tx_positions.txt`
- 用模型合成雙耳音訊
- 與真實的 `binaural.wav` 比較
- 計算每個 subject 的角度誤差

輸出示例：
```
Subject1: 平均誤差 5.2°
Subject2: 平均誤差 4.8°
Subject3: 平均誤差 6.1°
...
整體平均誤差: 5.4°
```

#### 2. 校準特定角度

```bash
# 用任意 mono 音訊校準 +60°
python calibrate_angle.py \
    --input dataset/testset_org/subject1/mono.wav \
    --desired 60 \
    --method itd \
    --save_best_wav calibrated_60deg.wav
```

**注意**：
- 結果是針對**該 mono 內容**的
- 不同的 mono 內容可能需要不同的 tx
- 但對於語音，差異通常不大

#### 3. 分析真實雙耳錄音

檢查不同 subject 的真實雙耳錄音的角度特性：

```python
import soundfile as sf
from src.doa import gcc_phat_estimate

for i in range(1, 8):
    bio, sr = sf.read(f'dataset/testset_org/subject{i}/binaural.wav')
    angle = gcc_phat_estimate(bio.T, sample_rate=sr)
    print(f'Subject{i}: {angle:+.2f}°')
```

這會顯示每個 subject 的真實雙耳錄音的平均方向。

#### 4. 主觀聽測（最重要）

```bash
# 合成不同角度的音訊
python synthesize.py \
    --input dataset/testset_org/subject1/mono.wav \
    --angle 60 \
    --output test_60deg.wav \
    --correction_mode segmented

# 用校準後的 tx
python synthesize.py \
    --input dataset/testset_org/subject1/mono.wav \
    --angle 60 \
    --output test_60deg_calibrated.wav \
    --angle_cache angle_tx_cache.json
```

戴耳機聽，主觀判斷：
- 方向感是否清晰？
- 是否符合預期角度？
- 校準前後有何差異？

### 常見誤解

#### ❌ 誤解 1：不同 subject 應該有不同的 mono.wav

**錯誤**：認為每個 subject 應該有不同的測試信號

**正確**：相同的測試信號才能公平比較不同 HRTF 的效果

#### ❌ 誤解 2：calibrate_angle.py 應該對不同 subject 產生不同結果

**錯誤**：期望相同的 mono.wav 在不同 subject 資料夾中產生不同結果

**正確**：相同的輸入必然產生相同的輸出（確定性系統）

#### ❌ 誤解 3：模型應該根據 subject 調整輸出

**錯誤**：認為模型會自動識別不同 subject 並調整 HRTF

**正確**：
- 預訓練模型使用固定的 HRTF（來自訓練數據）
- 不會根據輸入自動調整
- 如需個人化 HRTF，需要用該 subject 的數據 fine-tune

### testset vs testset_org 的差異

#### testset_org（原始）
- 未經角度補償的原始數據
- 用於測量模型的真實誤差
- 用於建立角度校正映射

#### testset（可能經過處理）
- 可能包含角度補償後的 tx_positions.txt
- 用於驗證校正效果
- 用於最終評估

### 總結

1. **所有 subject 的 mono.wav 相同是正常的**
   - 這是標準的 HRTF 測試設計
   - 用相同音源測試不同 HRTF

2. **calibrate_angle.py 對相同 mono.wav 產生相同結果是正常的**
   - 確定性系統的預期行為
   - 不是 bug

3. **正確的測試方法**
   - 用 evaluate.py 評估整體表現
   - 用 calibrate_angle.py 校準特定角度
   - 用主觀聽測驗證效果

4. **不同 subject 的差異體現在**
   - binaural.wav（真實雙耳錄音不同）
   - tx_positions.txt（頭部軌跡不同）
   - 不是 mono.wav（測試信號相同）

### 角度分布特徵

#### 1. 前方區域（-10° ~ +10°）- 最高密度
- 樣本數最多的區域
- 每個角度約 3,000-4,300 個樣本（0.41%-0.55%）

#### 2. 側前方區域（±10° ~ ±45°）- 高密度

- 左側（-10° ~ -45°）：略高於右側
- 右側（+10° ~ +45°）：分布較均勻

#### 3. 側方區域（±45° ~ ±90°）- 中等密度

- 左後方（-90° ~ -120°）：略高於右後方

## 三、使用情境釐清

### 權重與資料分布

- 若推論使用**未經 fine-tune 的原始權重**，其訓練資料來自 `dataset_original` 的**連續移動**軌跡
- 你現在要產生的是**固定方位角、靜態 `tx_positions`**，在分布上與訓練不完全一致
- 中偏側角（例如右側 **-60°** 或左側 **+60°**）更容易出現超過一般人 MAA 的誤差

### Fine-tune 考量

- 先前嘗試的 fine-tune 若效果不佳，**不應**把現有 `dataset/trainset` 當成目前誤差的主因
- 主因仍是「靜態角度輸出 + 預訓練 HRTF 與量測目標差異」的組合
- **分段校正**把 **|角| ≤ 45°** 交給 cubic 曲線，把 **|角| > 45°** 交給映射表線性內插
- 這可以避開 +60°~+90° 區間 **perceived 非單調** 時，單一 cubic 反向映射不穩定的問題
