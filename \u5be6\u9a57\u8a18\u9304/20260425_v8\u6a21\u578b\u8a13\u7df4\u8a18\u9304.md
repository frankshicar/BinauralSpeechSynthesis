## 一、設計動機

### 1.1 v7 的根本限制

v7 在現有 WarpNet 架構上做了 Stage 2 凍結 WaveNet 的改進，但有一個根本問題無法解決：

**WaveNet 在時域操作，無法直接感知頻域的相位結構。**

即使 Stage 2 凍結 WaveNet 讓 Warpnet 能無干擾地學習 Phase/IPD，Warpnet 的輸出仍然是時域 warpfield，Phase/IPD 的改善是間接的（透過 warp 影響相位），效率有限。

v6 實驗數據：
```
Stage 2 Phase 改善：0.969 → 0.800（-17.4%，60 epochs）
Stage 3 Phase 改善：0.800 → 0.800（幾乎停滯）
```

### 1.2 相關工作的啟發

**BinauralGrad（Leng et al., NeurIPS 2022）：**
- 核心洞見：將雙耳音訊分解為共同部分（ȳ）和差異部分（δ_L, δ_R）
- Stage 1 學共同部分，Stage 2 學差異部分，梯度完全分離
- 問題：使用 Diffusion model，推論需要 200 步

**DPATFNet（He et al., ICASSP 2025）：**
- 核心洞見：在 STFT 頻域直接處理 Phase/IPD，比時域更直接有效
- Phase-L2 達到 0.717（目前最佳，比 v6 的 0.800 改善 10.4%）
- 問題：L2 和 Phase loss 仍然一起訓練，存在梯度衝突

### 1.3 BinauralTFNet 的設計思路

結合兩篇論文的優點，用確定性模型取代 Diffusion：

```
Stage 1（時域）：CommonBranch 學兩耳共同部分 → L2 + DifferentiableITD 監督
Stage 2（頻域）：SpecificBranch 學左右耳差異 → Phase + IPD 監督（凍結 Stage 1）
Stage 3（端對端）：全部 fine-tune → L2×100 + Phase + IPD
```

## 二、架構設計

### 2.1 整體架構

```
輸入：mono (B×1×T), view (B×7×K)

┌─────────────────────────────────────────────────────┐
│  CommonBranch（Stage 1，時域）                        │
│                                                     │
│  Warpnet(mono, view) → warpfield (B×2×T)            │
│    warped = warp(mono, warpfield)  → B×2×T          │
│                                                     │
│  warpfield_common = mean(warpfield_L, warpfield_R)  │
│  warped_common = warp(mono, warpfield_common) B×1×T │
│  （warpfield 域平均，避免信號域平均的 comb filtering）│
│                                                     │
│  WaveNet(warped_common, view) → y_common (B×1×T)    │
│  監督：L2(y_common, mean(y_L_gt, y_R_gt))           │
│        + DifferentiableITD(warped, binaural) × 0.5  │
└─────────────────────────────────────────────────────┘
                    ↓ warped (B×2×T)
┌─────────────────────────────────────────────────────┐
│  SpecificBranch（Stage 2，頻域）                      │
│                                                     │
│  STFT(warped_L) → Y_warp_L (B×F×T_stft×2)          │
│  STFT(warped_R) → Y_warp_R (B×F×T_stft×2)          │
│                                                     │
│  DPAB(view) → cond (B×64×T_stft)                   │
│  TFResStack(Y_warp_L, Y_warp_R, cond)               │
│    → Y_delta_L, Y_delta_R (B×F×T_stft×2)           │
│                                                     │
│  iSTFT(Y_warp_L + Y_delta_L) → y_L (B×1×T)         │
│  iSTFT(Y_warp_R + Y_delta_R) → y_R (B×1×T)         │
│  （從 warped_L/R 出發，保留 Warpnet 的 ITD）         │
│  監督：Phase(y_L, y_L_gt) + Phase(y_R, y_R_gt)      │
│        + IPD(y_L, y_R, y_L_gt, y_R_gt)              │
└─────────────────────────────────────────────────────┘
```

### 2.2 關鍵設計決策

#### 決策 1：warpfield 域平均（非信號域平均）

CommonBranch 的 `warped_common` 用 warpfield 平均而非信號平均：

```python
# ✗ 錯誤：信號域平均 → comb filtering
warped_common = (warped_L + warped_R) / 2

# ✓ 正確：warpfield 域平均 → 物理意義是頭部中心延遲
warpfield_common = warpfield.mean(dim=1, keepdim=True)
warped_common = warp(mono, warpfield_common)
```

**原因：** 左右耳 ITD 差異在大角度時可達 0.8ms（38 samples @ 48kHz），信號域平均會造成梳狀濾波效應，高頻成分被嚴重壓制。

#### 決策 2：Stage 2 輸出端從 warped_L/R 出發

```python
# ✗ 原設計：Y_common 沒有 ITD，delta 要補太多
Y_L = Y_common + Y_delta_L

# ✓ 修改後：保留 Warpnet 的 ITD，delta 只修正 Phase 細節
Y_L = Y_warp_L + Y_delta_L
```

**原因：** ITD 主要由 Warpnet 的 warpfield 決定，在頻域用 TFResStack 重新學習 ITD 的效率不如時域 Warpnet。分工清晰：Warpnet 負責 ITD，TFResStack 負責 Phase 細節。

#### 決策 3：DifferentiableITD loss（Stage 1 新增）

現有的 `ITDLoss` 使用 `argmax`，不可微分，無法用於訓練。新增 `DifferentiableITDLoss`：

```python
# 用 GCC-PHAT 頻域相位比較，完全可微分
pred_cross = pred_L * pred_R.conj()
pred_cross = pred_cross / (pred_cross.abs() + 1e-8)  # GCC-PHAT 正規化
# 只用低頻段（< 1500 Hz），ITD 在低頻最有效
phase_diff = angle(pred_cross) - angle(tgt_cross)
loss = mean(atan2(sin(phase_diff), cos(phase_diff)) ** 2)
```

**原因：** 角度誤差的根本來源是 ITD，而 ITD 主要由 Stage 1 的 Warpnet 決定。直接監督 Warpnet 學習正確的 ITD，比靠 IPD loss 間接引導更有效。

#### 決策 4：Stage 2 輸入只用 Y_warp_L/R（4 channels）

```python
# ✗ 原設計：6 channels，Y_common 與 Y_warp_L/R 高度相關（冗餘）
Y_input = concat([Y_common, Y_warp_L, Y_warp_R])

# ✓ 修改後：4 channels，讓 TFResStack 自己學差異
Y_input = concat([Y_warp_L, Y_warp_R])
```

#### 決策 5：Loss 權重基於實際數值範圍設定

v7 實驗數據顯示各 loss 的真實數值範圍：
```
L2    ~ 0.0001  （比 Phase/IPD 小約 8000 倍）
Phase ~ 0.80
IPD   ~ 0.62
```

基於此設定各 Stage 的 weight：
- Stage 1：L2（無 weight）+ ITD × 0.5
- Stage 2：Phase × 1.0 + IPD × 1.0（量級相近，不需要調整）
- Stage 3：L2 × 100 + Phase × 1.0 + IPD × 1.0（L2 貢獻約 1.8%，不壓制 Phase）

## 三、訓練配置

### 3.1 三階段策略

| 階段 | Epochs | 可訓練參數 | Loss | 目標 |
|------|--------|-----------|------|------|
| Stage 1 | 0–80 | CommonBranch（全部）| L2 + ITD×0.5 | 學好共同部分，監督 Warpnet ITD |
| Stage 2 | 80–160 | SpecificBranch（凍結 CommonBranch）| Phase + IPD | 頻域直接優化 Phase/IPD |
| Stage 3 | 160–180 | 全部 | L2×100 + Phase + IPD | 修正 iSTFT 重建誤差 |

### 3.2 Loss 函數

```python
# Stage 1
loss_s1 = L2Loss(y_common, mean(y_L_gt, y_R_gt))
        + DifferentiableITDLoss(warped, binaural) * 0.5

# Stage 2（Phase/IPD 量級相近，weight 設 1.0）
loss_s2 = PhaseLoss(y_L, y_L_gt) + PhaseLoss(y_R, y_R_gt) + IPDLoss(y_L, y_R, y_L_gt, y_R_gt)

# Stage 3（L2×100 讓 L2 貢獻約 1.8%，不壓制 Phase）
loss_s3 = (L2Loss(y_L, y_L_gt) + L2Loss(y_R, y_R_gt)) * 100.0
        + PhaseLoss(y_L, y_L_gt) + PhaseLoss(y_R, y_R_gt)
        + IPDLoss(y_L, y_R, y_L_gt, y_R_gt)
```

### 3.3 LR 調度

```python
CosineAnnealingWarmRestarts(T_0=80, eta_min=5e-6)
```

每個 Stage 切換時重建 scheduler，各自從 lr=3e-4 開始 cosine 衰減。Stage 3 的初始 LR 限制為 min(當前 LR, 3e-4)。
