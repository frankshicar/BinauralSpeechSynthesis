## 關鍵修改摘要 (Key Changes Summary)

### 🆕 IPD Loss 實作 (2026-04-11)

**問題 (Problem)**：
1. ITD 系統性偏大（±90° 超出物理極限 ±800μs）
2. 左右不對稱（右側誤差比左側大）
3. 大部分角度不符合復健應用標準

**解決方案 (Solution)**：
加入 IPD (Interaural Phase Difference) Loss 來直接約束雙耳相位差

**修改檔案 (Modified Files)**：
- `src/losses.py`: 新增 `IPDLoss` 類別
- `src/trainer.py`: 整合 IPD loss 到訓練流程
- `train.py`: 設定 IPD loss 權重為 0.1

**詳細說明**：參考 `IPD_Loss_說明.md`

### 🔧 噪音修復 (Noise Fix)

**問題 (Problem)**：輸出音訊有噪音
**原因 (Cause)**：

1. 缺少 `clamp()` 操作 → 音訊值超出 [-1, 1] 範圍
2. 錯誤的 Context Padding 實作 → 邊界不連續

**修復 (Fix)**：

```python
# evaluate.py 的 chunked_forwarding()
# 恢復原始實作 + 加上 clamp
binaural = th.clamp(binaural, min=-1, max=1).cpu()
```

**調整 IPD 權重：**

編輯 `train.py` 第 52 行：

"loss_weights": {"l2": 1.0, "phase": 0.01, "ipd": 0.1}

- 起始值：`ipd: 0.1`

## 疑難排解 (Troubleshooting)

### 問題：IPD Loss 不下降

1. 檢查 IPD 權重是否太小：
   ```python
   # train.py
   "loss_weights": {"l2": 1.0, "phase": 0.01, "ipd": 0.1}
   ```
2. 增加 IPD 權重到 0.2 或 0.3
3. 確認其他 loss 是否正常下降

### 問題：訓練後聲音品質下降

1. IPD 權重可能太大
2. 降低 IPD 權重到 0.05 或 0.08
3. 增加訓練 epoch 數

### 問題：左右不對稱仍然存在

1. 檢查訓練資料的角度分布
2. 考慮資料增強（左右翻轉）
3. 增加 IPD 權重

### 問題：仍然有噪音

1. 確認使用的是修復後的版本：

   ```bash
   git log --oneline -1
   # 應該顯示: bc4184b [2026-01-26] 程式碼整理與噪音修復
   ```

2. 檢查輸出音訊的振幅是否在 [-1, 1] 範圍內
3. 嘗試回到原始版本對比

### 問題：評估指標異常

1. 確認模型檔案路徑正確
2. 確認測試資料集路徑正確
3. 檢查 GPU 記憶體是否足夠
4. 查看詳細錯誤訊息

### 問題：找不到模型檔案

- 確認已完成訓練，或下載預訓練模型
- 檢查 `--model_file` 參數路徑
