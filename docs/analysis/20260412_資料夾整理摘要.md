## 整理目標

## 整理前狀態

### 根目錄檔案混亂
根目錄包含 **40+ 個檔案**，類型混雜：
- Python 腳本（核心 + 測試 + 除錯）
- Markdown 文檔（技術文檔 + 說明 + 記錄）
- Shell 腳本（訓練 + 工具）
- 文字日誌（診斷 + 記錄）
- 資料資料夾（KEMAR data）

### 問題
1. **檔案類型混雜**：難以快速找到需要的檔案
2. **功能不清晰**：核心功能與實驗工具混在一起
3. **維護困難**：新增檔案不知道放哪裡
4. **結構混亂**：沒有明確的組織邏輯

## 新增檔案規則

### 檔案類型對應資料夾

| 檔案類型 | 目標資料夾 | 範例 |
|----------|------------|------|
| 核心功能腳本 | 根目錄 | `new_core_feature.py` |
| 實驗腳本 | `experiments/` | `test_new_method.py` |
| 技術文檔 | `docs/` | `NEW_FEATURE_GUIDE.md` |
| Shell 腳本 | `scripts/` | `start_new_experiment.sh` |
| 日誌檔案 | `logs/` | `experiment_results.txt` |
| 實驗記錄 | `實驗記錄/` | `20260412_新實驗.md` |

### 命名建議

#### 實驗腳本
- 測試：`test_*.py`
- 診斷：`diagnose_*.py`, `check_*.py`
- 校正：`calibrate_*.py`
- 比較：`compare_*.py`

#### 文檔檔案
- 技術指南：`*_GUIDE.md`
- 問題修復：`*_FIX.md`
- 功能升級：`*_UPGRADE.md`
- 系統總結：`*_SUMMARY.md`

#### 腳本檔案
- 啟動：`start_*.sh`
- 恢復：`resume_*.sh`
- 驗證：`verify_*.sh`

1. **每月檢查**：

3. **標籤管理**：重要版本打 Git 標籤
